# 任意数量量子位的 W 状态

**输入：** $\ket{0 \dots 0}$ 状态下的 $N$ 量子位（$N$ 不一定是 2 的幂）。

**目标：** 将量子位的状态更改为 [W 状态](https://en.wikipedia.org/wiki/W_state) - $N$ 基础状态在汉明权重为 1 的 $N$ 量子位上的相等叠加。

例如，对于 $N = 3$，所需状态为 $\frac{1}{\sqrt{3}}\big(\ket{100} + \ket{010} + \ket{001}\big)$。

## 解决方案注释

这个问题允许有多种解决方案，这些解决方案依赖于从任意旋转到递归到后选择的技术。

### 迭代解决方案

第一种方法依赖于执行一系列受控旋转。

要在单个量子位上准备加权叠加 $\cos \theta \ket{0} + \sin \theta \ket{1}$，您需要从 $\ket{0}$ 状态开始，并对其应用 $R_y$ 门，角度参数等于 $2 \theta$。 
您将角度为 $2 \theta_1 = 2\arcsin \frac{1}{\sqrt{N}}$ 的 $R_y$ 门应用于寄存器的第一个量子位，以准备以下状态：

$$(\cos \theta_1 \ket{0} + \sin \theta_1 \ket{1}) \otimes \ket{0 \dots 0} = \frac{1}{\sqrt{N}}\ket{10 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}}\ket{00 \dots 0} $$

第一项 $\frac{1}{\sqrt{N}}\ket{10 \dots 0}$ 已与 $\ket{W_N}$ 状态的第一项匹配；现在您需要将第二项 $\frac{\sqrt{N-1}}{\sqrt{N}}\ket{00 \dots 0}$ 转换为其余的 $\ket{W_N}$ 项。

要准备与 $\ket{W_N}$ 状态的第二项相匹配的项，您可以将另一个 $R_y$ 门应用于项 $\ket{00 \dots 0}$，这次是第二个量子位，角度为 $2 \theta_2 = 2\arcsin \frac{1}{\sqrt{N-1}}$。
为了确保它不会影响您已经满意的术语，您将应用 $R_y$ 门的受控版本，以状态 $\ket{0}$ 的寄存器的第一个量子位作为控制。
这会将状态更改为

$$\frac{1}{\sqrt{N}}\ket{10 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}} \ket{0} \otimes (\cos \theta_2 \ket{0} + \sin \theta_2 \ket{1}) \otimes \ket{0 \dots 0} = $$
$$= \frac{1}{\sqrt{N}}\ket{10 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}} \frac{1}{\sqrt{N-1}} \ket{010 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}} \frac{\sqrt{N-2}}{\sqrt{N-1}} \ket{000 \dots 0}$$

现在您有两项与 $\ket{W_N}$ 状态的项相匹配，并且需要将第三项 $\frac{\sqrt{N-2}}{\sqrt{N}}\ket{00 \dots 0}$ 转换为其余项。

您将继续这样操作，一次准备 $\ket{W_N}$ 状态的一项，直到最后一个量子位上的旋转将成为 $X$ 门，控制处于 $\ket{0 \dots 0}$ 状态的所有先前量子位。

来源：SolutionA.qs

```qsharp
namespace Kata {
    import Std.Convert.*;
    import Std.Math.*;

    operation WState_Arbitrary(qs : Qubit[]) : Unit is Adj + Ctl {
        let N = Length(qs);
        Ry(2.0 * ArcSin(Sqrt(1.0 / IntAsDouble(N))), qs[0]);
        for i in 1..N - 1 {
            ApplyControlledOnInt(
                0,
                Ry(2.0 * ArcSin(Sqrt(1.0 / IntAsDouble(N - i))), _),
                qs[0..i - 1],
                qs[i]
            );
        }
    }
}
```

### 递归解决方案

如果您注意到，您可以使用递归来表达相同的门序列

$$\ket{W_N} = \frac{1}{\sqrt{N}}\ket{10 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}}\ket{0} \otimes \ket{W_{N-1}}$$

该解决方案的第一步仍然是将角度为 $2 \theta_1 = 2\arcsin \frac{1}{\sqrt{N}}$ 的 $R_y$ 门应用于寄存器的第一个量子位，以准备以下状态：

$$\frac{1}{\sqrt{N}}\ket{10 \dots 0} + \frac{\sqrt{N-1}}{\sqrt{N}}\ket{00 \dots 0} $$

但是您可以将其余的受控旋转表示为准备 $\ket{W_{N-1}}$ 状态的操作，并在第一个量子位的 $\ket{0}$ 状态上进行控制。

> 请注意，您不必自己实现此操作的受控版本；将 `is Adj + Ctl` 添加到操作 `WState_Arbitrary` 的签名中即可指定必须自动生成受控变体。

来源：SolutionB.qs

```qsharp
namespace Kata {
    import Std.Convert.*;
    import Std.Math.*;

    operation WState_Arbitrary(qs : Qubit[]) : Unit is Adj + Ctl {
        let N = Length(qs);
        Ry(2.0 * ArcSin(Sqrt(1.0 / IntAsDouble(N))), qs[0]);
        if N > 1 {
            ApplyControlledOnInt(0, WState_Arbitrary, [qs[0]], qs[1...]);
        }
    }
}
```

### 选择后解决方案

假设您知道如何为 $N = 2^k$ 准备 $W$ 状态（请参阅上一个任务），并弄清楚如何使用这些知识作为解决此任务的构建块。

让我们看看 $N \neq 2^k$: $N = 3$ 的最小可能情况（您将能够将这种情况的解决方案推广到任意数量的量子位）。目标 $W$ 状态如下所示：

$$\ket{W_3} = \frac{1}{3}\big(\ket{100} + \ket{010} + \ket{001}\big)$$

您首先要找到大于或等于 $N$ 的 2 $P$ 的最小幂；对于这种情况$N = 3$，因此该功率将为$P = 4$。您将分配额外的 $P - N$ 量子位，并使用上一个任务的解决方案来准备如下所示的 $W_P$ 状态（额外量子位的状态以粗体突出显示）：

$$\ket{W_4} = \frac{1}{2}\big( \ket{100\textbf{0}} + \ket{010\textbf{0}} + \ket{001\textbf{0}} + \ket{000\textbf{1}} \big) = $$
$$= \frac{\sqrt3}{2} \cdot \frac{1}{\sqrt3}\big(\ket{100} + \ket{010} + \ket{001} \big) \otimes \ket{\textbf{0}} + \frac{1}{2}\ket{000} \otimes \ket{\textbf{1}} = $$
$$= \frac{\sqrt3}{2} \ket{W_3} \otimes \ket{\textbf{0}} + \frac{1}{2}\ket{000} \otimes \ket{\textbf{1}}$$

如您所见，如果额外的量子位处于 $\ket{0}$ 状态，则您关心的主要 3 个量子位处于正确的 $\ket{W_3}$ 状态。

如果您只测量额外的量子位，会发生什么？这会导致系统部分崩溃到测量结果定义的状态：
* 如果结果是 $\ket{0}$，系统崩溃到 $\ket{W_3}$ 状态 - 这正是您想要实现的。
* 如果结果是$\ket{1}$，系统崩溃到状态$\ket{000}$，所以你的目标没有实现。好消息是，这只发生在 $25\%$ 的情况下，您可以重试。

如果将此方法推广到任意 $N$，您将得到

$$\ket{W_P} = \frac{\sqrt{N}}{\sqrt{P}} \ket{W_N} \otimes \ket{\textbf{0}}^{\otimes P-N} + \frac{\sqrt{P-N}}{\sqrt{P}} \ket{0}^{\otimes N} \otimes \ket{W_{P-N}}$$

测量额外的 $P-N$ 量子位为您提供了两种可能性：
* 所有额外的量子位都处于$\ket{0}$状态；这意味着主量子位崩溃到 $\ket{W_N}$ 状态。 
* 额外量子位之一处于 $\ket{1}$ 状态；这意味着主量子位崩溃到 $\ket{0}^{\otimes N}$ 状态，这不是**所需的状态。在这种情况下，您将重置并重试，直到所有额外的量子位都处于 $\ket{0}$ 状态。

来源：SolutionC.qs

```qsharp
namespace Kata {
    operation WState_Arbitrary (qs : Qubit[]) : Unit {
        let N = Length(qs);
        if N == 1 {
            X(qs[0]);
        } else {
            mutable P = 1;
            for i in 1 .. 6 {
                if P < N {
                    set P *= 2;
                }
            }

            use anc = Qubit[P - N];
            mutable allZeros = true;
            repeat {
                WState_PowerOfTwo(qs + anc);

                set allZeros = true;
                for i in 0 .. (P - N) - 1 {
                    if MResetZ(anc[i]) == One {
                        set allZeros = false;
                    }
                }
            }
            until (allZeros);
        }
    }

    operation WState_PowerOfTwo (qs : Qubit[]) : Unit is Adj + Ctl {
        let N = Length(qs);

        if N == 1 {
            X(qs[0]);
        } else {
            let K = N / 2;
            use anc = Qubit();
            H(anc);

            ApplyControlledOnInt(0, WState_PowerOfTwo, [anc], qs[0 .. K - 1]);
            ApplyControlledOnInt(1, WState_PowerOfTwo, [anc], qs[K .. N - 1]);

            for i in K .. N - 1 {
                CNOT(qs[i], anc);
            }
        }
    }
}
```
