# 四位串

**输入：**

1. $N$ 个量子位处于 $\ket{0 \dots 0}$ 状态。
2. 四个位串表示为 `Bool[][]` `bits`。

`bits` 是一个大小为 $4 \times N$ 的数组，其描述位串如下：
    - `bits[i]` 描述第 *i* 个位串并具有 $N$ 个元素。
    - 所有四个位串都是不同的。

**目标：** 创建由位串给出的四个基本状态的相等叠加。

> 例如，对于$N = 3$和`bits =  [[false, true, false], [true, false, false], [false, false, true], [true, true, false]]`，您需要准备的状态是$\frac{1}{2} \big(\ket{010} + \ket{100} + \ket{001} + \ket{110}\big)$。

## 解决方案注释

您将使用与上一个任务中使用的相同的辅助量子位技巧。

由于所需的叠加有 4 个幅度相等的基态，因此您将需要两个量子位来定义唯一的基来控制叠加中每个基态的准备。

首先分配两个额外的量子位，并通过对每个量子位应用 $H$ 门来准备它们上所有 2 量子位状态的相等叠加：

$$\frac12 (\ket{00} + \ket{01} + \ket{10} + \ket{11})_a \otimes \ket{0 \dots 0}_r$$

然后，对于四个给定的位串中的每一个，您都可以遍历它并在量子位的主寄存器上准备匹配的基础状态，使用受控的 $X$ 门和辅助量子位的相应基础状态作为控制。

例如，在准备位串 `bits[0]` 时，您可以应用在基础状态 $\ket{00}$ 上控制的 $X$ 门；当准备位串 `bits[1]` 时，您可以应用 $\ket{10}$ 上控制的 $X$ 门，依此类推。

> 您可以选择用作控制的 2 量子位基础状态与主寄存器上准备的位串的任意匹配。
> 由于所有幅度都相同，因此结果不取决于哪个状态控制哪个位串准备。
> 使用转换为小尾数的位串索引来控制位串的准备可以很方便。
> Q# 库函数 [`ApplyControlledOnInt`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.canon/applycontrolledonint) 正是这样做的。

此后系统将处于以下状态

$$\frac12 (\ket{00}_a \ket{bits_0}_r + \ket{10}_a \ket{bits_1}_r + \ket{01}_a \ket{bits_2}_r + \ket{11}_a \ket{bits_3}_r)$$

作为最后一步，您必须取消计算辅助量子位，即将它们返回到 $\ket{00}$ 状态，以将它们从主寄存器中解开。

与上一个任务中相同，您将使用 [`ApplyControlledOnBitString`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.canon/applycontrolledonbitstring) 以及相应的位字符串和 `X` 操作作为参数，量子寄存器作为控制，辅助量子位作为目标。

您将分别取消计算它们中的每一个，因此其中一个辅助量子位将使用 `bits[1]` 和 `bits[3]` 位字符串作为控制来取消计算，另一个辅助量子位将使用 `bits[2]` 和 `bits[3]` 作为控制来取消计算。

来源：SolutionA.qs

```qsharp
namespace Kata {
    operation FourBitstringSuperposition (qs : Qubit[], bits : Bool[][]) : Unit {
        use anc = Qubit[2];
        // Put two ancillas into equal superposition of 2-qubit basis states
        ApplyToEachA(H, anc);

        // Set up the right pattern on the main qubits with control on ancillas
        for i in 0 .. 3 {
            for j in 0 .. Length(qs) - 1 {
                if bits[i][j] {
                    ApplyControlledOnInt(i, X, anc, qs[j]);
                }
            }
        }

        // Uncompute the ancillas, using patterns on main qubits as control
        for i in 0 .. 3 {
            if i % 2 == 1 {
                ApplyControlledOnBitString(bits[i], X, qs, anc[0]);
            }
            if i / 2 == 1 {
                ApplyControlledOnBitString(bits[i], X, qs, anc[1]);
            }
        }
    }
}
```

或者，您可以利用 Q# 的递归功能来创建四个位字符串的叠加。  该解决方案还可以扩展到任意数量的位串，而无需更改代码。

在此过程中，您将查看每个字符串的第一位，并根据答案的第一个量子位相应地调整测量 $\ket{0}$ 或 $\ket{1}$ 的概率。然后，您将再次递归调用（根据需要）该过程，以根据第一位调整第二位的测量概率。  该过程不断重复，直到不再提供输入位为止。

例如，考虑以下要在其上创建叠加的四个位串：$\ket{001}, \ket{101}, \ket{111}, \ket{110}$。

您可以将需要准备的叠加状态重写为

$$\frac12 \big(\ket{001} + \ket{101} + \ket{111} + \ket{110} \big) = \frac12 \ket{0} \otimes \ket{01} + \frac{\sqrt3}{2} \ket{1} \otimes \frac{1}{\sqrt3} \big(\ket{10} + \ket{11} + \ket{10} \big)$$

作为解决方案的第一步，您需要在第一个量子位上准备一个状态 $\frac12 \ket{0} + \frac{\sqrt3}{2} \ket{1}$（以 $\frac14$ 概率测量 $\ket{0}$，以 $\frac34$ 概率测量 $\ket{1}$。为此，您将对第一个量子位应用 [`Ry`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.intrinsic/ry) 旋转。

此后，您需要根据第一个量子位的状态将其余量子位准备在适当的状态 - 如果第一个量子位处于状态 $\ket{0}$，则状态 $\ket{01}$；如果第一个量子位处于状态 $\ket{1}$，则状态 $\frac{1}{\sqrt3} \big(\ket{10} + \ket{11} + \ket{10} \big)$。您可以使用相同的逻辑递归地执行此操作。让我们详细介绍一下这个示例。

递归的第二个量子位的情况类似，但取决于第一个量子位。如果第一个量子位测量 $\ket{0}$，那么您希望第二个量子位以 $100\%$ 概率测量 $\ket{0}$，但如果它测量 $\ket{1}$，您希望它以 $\frac13$ 概率测量 $\ket{0}$，以 $\ket{1}$ 概率测量 $\ket{1}$ $\frac23$ 概率。  为此，您可以以第一个量子位作为控制，对第二个量子位进行受控 [`Ry`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.intrinsic/ry) 旋转。

本例中的第三个量子位将具有三种情况，因为它取决于前两个量子位；这是从递归中自然得出的。

1. 如果前两个量子位测量 $\ket{00}$，则您需要第三个量子位以 $100\%$ 概率测量 $\ket{0}$。
2. 如果前两个量子位测量 $\ket{10}$，则您需要第三个量子位以 $100\%$ 概率测量 $\ket{1}$。
3. 如果前两个量子位测量 $\ket{11}$，则您需要第三个量子位以 $\frac12$ 概率测量 $\ket{0}$，以 $\frac12$ 概率测量 $\ket{1}$。  正如第二个量子位一样，第三个量子位上的受控 [`Ry`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.intrinsic/ry) 旋转将实现此目标。

> 您将使用 [`ApplyControlledOnBitString`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.canon/applycontrolledonbitstring) 操作根据先前几个量子位的状态执行旋转。

来源：SolutionB.qs

```qsharp
namespace Kata {
    import Std.Convert.*;
    import Std.Math.*;

    operation FourBitstringSuperposition(qs : Qubit[], bits : Bool[][]) : Unit {
        FourBitstringSuperposition_Recursive([], qs, bits);
    }

    operation FourBitstringSuperposition_Recursive(currentBitString : Bool[], qs : Qubit[], bits : Bool[][]) : Unit {
        // an array of bit strings whose columns we are considering begin with |0⟩
        mutable zeroLeads = [];
        // an array of bit strings whose columns we are considering begin with |1⟩
        mutable oneLeads = [];
        // the number of bit strings we're considering
        let rows = Length(bits);
        // the current position we're considering
        let currentIndex = Length(currentBitString);

        if rows >= 1 and currentIndex < Length(qs) {
            // figure out what percentage of the bits should be |0⟩
            for row in 0..rows - 1 {
                if bits[row][currentIndex] {
                    set oneLeads = oneLeads + [bits[row]];
                } else {
                    set zeroLeads = zeroLeads + [bits[row]];
                }
            }
            // rotate the qubit to adjust coefficients based on the previous bit string
            // for the first path through, when the bit string has zero length,
            // the Controlled version of the rotation will perform a regular rotation
            let theta = ArcCos(Sqrt(IntAsDouble(Length(zeroLeads)) / IntAsDouble(rows)));
            ApplyControlledOnBitString(
                currentBitString,
                Ry,
                qs[0..currentIndex - 1],
                (2.0 * theta, qs[currentIndex])
            );

            // call state preparation recursively based on the bit strings so far
            FourBitstringSuperposition_Recursive(currentBitString + [false], qs, zeroLeads);
            FourBitstringSuperposition_Recursive(currentBitString + [true], qs, oneLeads);
        }
    }
}
```
