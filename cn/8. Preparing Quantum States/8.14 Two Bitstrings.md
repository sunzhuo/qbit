# 两个位串

**输入：**

1. $N$ ($N \ge 1$) 个量子位处于 $\ket{0 \dots 0}$ 状态。
2. 长度为 $N$ 的两个位串表示为 `Bool[]`。位值`false`和`true`对应于$\ket{0}$和$\ket{1}$状态。您可以保证这两个位串至少有一位不同。

**目标：** 将量子位的状态更改为位串给出的基本状态的相等叠加。

> 例如，对于位串 `[false, true, false]` 和 `[false, false, true]`，所需的状态是 $\frac{1}{\sqrt{2}}\big(\ket{010} + \ket{001}\big)$。

## 解决方案注释

上一个任务中描述的使用辅助量子位来控制准备过程的策略也可以应用于此任务。

您将首先分配一个辅助量子位，并使用 $H$ 门将其准备在 $\frac{1}{\sqrt2} (\ket{0} + \ket{1})$ 状态。系统的整体状态将是

$$\frac{1}{\sqrt2} (\ket{0} + \ket{1})_a \otimes \ket{0 \dots 0}_r = \frac{1}{\sqrt2} (\ket{0}_a \otimes \ket{0 \dots 0}_r + \ket{1}_a \otimes \ket{0 \dots 0}_r)$$

此时，您可以逐位分别准备目标状态的两个基本状态，控制其中一个在辅助量子位的 $\ket{0}$ 状态上的准备，以及另一个在 $\ket{1}$ 状态上的准备。 
如果其中一个位串中的一位是 `true`，则您将应用受控 $X$ 门，以辅助量子位作为控制，寄存器相应位置中的量子位作为目标，并根据您当前考虑的位串将其控制在 $\ket{0}$ 或 $\ket{1}$ 状态。 
这种受控门可以使用 [`ApplyControlledOnInt`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.canon/applycontrolledonint) 库函数来实现。

之后系统的状态将是
$$\frac{1}{\sqrt2} (\ket{0}_a \otimes \ket{bits_1}_r + \ket{1}_a \otimes \ket{bits_2}_r)$$

最后，您将使用 [`ApplyControlledOnBitString`](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.canon/applycontrolledonbitstring) 库函数来取消计算辅助量子位，其中第二个位字符串和 `X` 操作作为参数，量子寄存器作为控制，辅助量子位作为目标。 
这只会影响 $\ket{1}_a \otimes \ket{bits_2}_r$ 项，翻转其中辅助量子位的状态并使系统达到最终状态：

$$\ket{0}_a \otimes \frac{1}{\sqrt2} (\ket{bits_1} + \ket{bits_2})_r$$

来源：SolutionA.qs

```qsharp
namespace Kata {
    operation TwoBitstringSuperposition (qs : Qubit[], bits1 : Bool[], bits2 : Bool[]) : Unit is Adj + Ctl {
        use q = Qubit();
        H(q);

        for i in 0 .. Length(qs) - 1 {
            if bits1[i] {
                ApplyControlledOnInt(0, X, [q], qs[i]);
            }
            if bits2[i] {
                ApplyControlledOnInt(1, X, [q], qs[i]);
            }
        }

        // uncompute the auxiliary qubit to release it
        ApplyControlledOnBitString(bits2, X, qs, q);
    }    
}
```

如果您在此角色中使用寄存器中的量子位之一，也可以在不使用额外量子位的情况下解决任务。 
在遍历寄存器和位串时，第一次位串不一致时，相应位置的量子位将扮演辅助量子位的角色；您可以使用 $H$ 门将其叠加，并使用该量子位作为控制来执行所有后续位翻转。

这可以为您节省一个额外的量子位，并允许跳过未计算的步骤，尽管代码变得不太优雅。 
您将移动比较两个位字符串以查找它们与函数 `FindFirstDiff` 不同的第一个位置的经典逻辑。

来源：SolutionB.qs

```qsharp
namespace Kata {
    function FindFirstDiff (bits1 : Bool[], bits2 : Bool[]) : Int {
        for i in 0 .. Length(bits1) - 1 {
            if bits1[i] != bits2[i] {
                return i;
            }
        }
        return -1;
    }

    operation TwoBitstringSuperposition (qs : Qubit[], bits1 : Bool[], bits2 : Bool[]) : Unit is Adj + Ctl {
        // find the index of the first bit at which the bit strings are different
        let firstDiff = FindFirstDiff(bits1, bits2);

        // Hadamard corresponding qubit to create superposition
        H(qs[firstDiff]);

        // iterate through the bit strings again setting the final state of qubits
        for i in 0 .. Length(qs) - 1 {
            if bits1[i] == bits2[i] {
                // if two bits are the same, apply X or nothing
                if bits1[i] {
                    X(qs[i]);
                }
            } else {
                // if two bits are different, set their difference using CNOT
                if i > firstDiff {
                    CNOT(qs[firstDiff], qs[i]);
                    if bits1[i] != bits1[firstDiff] {
                        X(qs[i]);
                    }
                }
            }
        }
    }  
}
```
