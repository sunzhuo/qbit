# 零和位串

**输入：**

1. $N$ ($N \ge 1$) 个量子位处于 $\ket{0 \dots 0}$ 状态。
2. 长度为 $N$ 的位串，表示为 `Bool[]`。位值`false`和`true`对应于$\ket{0}$和$\ket{1}$状态。您可以保证位串的第一位是 `true`。

**目标：** 将量子位的状态更改为 $\ket{0 \dots 0}$ 和位串给出的基础状态的相等叠加。

> 例如，对于位串 `[true, false]`，所需的状态是 $\frac{1}{\sqrt{2}}\big(\ket{00} + \ket{10}\big)$。

## 解决方案注释

> 在量子位寄存器中准备叠加状态的常见策略是使用辅助量子位（或多个，用于更复杂的状态）。辅助量子位可以通过应用 Hadamard 门的常用方法进入叠加状态（或绕 $Y$ 轴旋转以实现不均匀叠加）。 
> 然后，通过将辅助量子位状态用作 $CNOT$ 门的控制量子位，根据辅助量子位状态单独准备所需叠加的基础状态。将准备在辅助状态的 $\ket{0}$ 组件上控制其中一个基本状态，而另一个在 $\ket{1}$ 组件上控制。 
> 最后，您必须通过取消计算辅助量子位将其返回到 $\ket{0}$ 状态，即使用从 $\ket{1}$ 组件准备的基础状态作为以辅助量子位为目标的 $CNOT$ 门的控制量子位。
>
> 有关使用此方法的更多详细信息，请参阅“四个位串的叠加”和“$2^k$ 量子位上的 W 状态”任务的解决方案。但是，对于此任务，您可以想出一个更简单的解决方案。 
> 您可以使用寄存器中的第一个量子位来实现此目的，而不是分配新的量子位用作辅助，因为可以保证构成所需叠加的两个基本向量中的第一位是不同的。
> 这使您无需分配新的量子位，并让我们跳过非计算步骤，因为充当后续准备步骤控制的量子位是所需结果的一部分。

考虑此 kata 中要求准备 Bell 状态和 GHZ 状态的早期任务；该任务中叠加状态的结构是这些场景的更一般情况：所有这些场景都要求准备两个不同基础状态的相等叠加。

解决方案的第一步与这些任务中的相同：使用 $H$ 门将寄存器中的第一个量子位放入 $\ket{0}$ 和 $\ket{1}$ 的相等叠加中，以获得以下状态：

$$\frac{1}{\sqrt2} (\ket{0} + \ket{1}) \otimes \ket{0 \dots 0} = \frac{1}{\sqrt2} (\ket{00 \dots 0} + \ket{10 \dots 0})$$

叠加的第一项已经与所需状态匹配，因此您需要修复第二项。
为此，您将遍历寄存器中的剩余量子位，检查位串 `bits` 相应位置中的位是否为 `true`。 
如果是，则需要在叠加的第二项中将该量子位的状态从 $0$ 调整到 $1$（并在第一项中保持不变）。 
您可以使用 $CNOT$ 门来进行此更改，其中第一个量子位作为控制，当前量子位作为目标。
当您完成像这样浏览寄存器时，寄存器将处于所需的叠加状态。

来源：Solution.qs

```qsharp
namespace Kata {
    operation ZeroAndBitstringSuperposition (qs : Qubit[], bits : Bool[]) : Unit is Adj + Ctl {
        H(qs[0]);

        for i in 1 .. Length(qs) - 1 {
            if bits[i] {
                CNOT(qs[0], qs[i]);
            }
        }
    }
}
```
