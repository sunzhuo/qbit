# 格林伯格-霍恩-泽林格州

**输入：** $N$ ($N \ge 1$) 处于 $\ket{0 \dots 0}$ 状态的量子位（存储在长度为 $N$ 的数组中）。

**目标：** 将量子位的状态更改为 GHZ 状态 $\frac{1}{\sqrt{2}} \big (\ket{0\dots0} + \ket{1\dots1}\big)$。

## 解决方案注释

单量子位 GHZ 状态是您在第一个任务中讨论的加状态 $\frac{1}{\sqrt{2}} \big (\ket{0} + \ket{1}\big)$。提醒一下，该状态是通过应用 Hadamard 门来准备的。

2 量子位 GHZ 状态是我们在前面两个任务中讨论过的贝尔状态 $\frac{1}{\sqrt{2}} \big (\ket{00} + \ket{11}\big)$。

下一个是 3 量子位 GHZ 状态：
$$\ket{GHZ} = \frac{1}{\sqrt{2}} \big (\ket{000} + \ket{111}\big)$$

让我们使用 2 量子位状态作为构建块来构建 3 量子位的状态。首先，我们将第三个量子位添加到上述状态（在前两个量子位的右侧）。
将此状态与所需的最终状态进行比较，您会发现它们仅在第三个（最右侧）量子位上有所不同：

$$\ket{\Phi^+} \ket{0} = \frac{1}{\sqrt{2}} \big (\ket{000} + \ket{11\textbf{0}}\big)$$
$$\ket{GHZ} = \frac{1}{\sqrt{2}} \big (\ket{000} + \ket{11\textbf{1}}\big)$$

使用第一个（最左边）量子位作为控制位和第三个（最右边）量子位作为目标量子位来应用受控 $NOT$ 操作可以修复此差异。

因此，您可以得出通用解决方案：将 Hadamard 门应用于第一个量子位，并以第一个量子位作为控制、其他每个量子位作为目标执行一系列 $CNOT$ 门。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Arrays.*;

    operation GHZ_State(qs : Qubit[]) : Unit is Adj + Ctl {
        H(qs[0]);

        // Library function Rest returns all array elements except for the first one
        for q in Rest(qs) {
            CNOT(qs[0], q);
        }
    }
}
```
