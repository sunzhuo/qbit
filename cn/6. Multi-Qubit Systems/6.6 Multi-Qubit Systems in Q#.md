# Q# 中的多量子位系统

此演示向您展示如何在 Q# 中分配多个量子位并检查它们的联合状态。它使用单量子位门来操纵各个量子位状态 - 如果您需要复习它们，请查看单量子位门套路。

这些演示使用函数 `DumpMachine` 来打印量子模拟器的状态。
处理多量子位系统时，`DumpMachine` 打印有关具有非零幅度的每个基本状态的信息，每行一个基本状态，与单量子位系统相同。
基本状态表示为位串，每个分配的量子位一位，最左边的位对应
分配给最早分配的量子位。 （如果量子位作为数组立即分配，则最左边的位对应
到数组的第一个元素。）

来源：MultiQubitSystems.qs

```qsharp
namespace Kata {
    import Std.Diagnostics.*;

    @EntryPoint()
    operation MultiQubitSystemsDemo() : Unit {
        // This allocates an array of 2 qubits, each of them in state |0⟩.
        // The overall state of the system is |00⟩.
        use qs = Qubit[2];
        // X gate changes the first qubit into state |1⟩.
        X(qs[0]);
        Message("The system in now in state |10⟩:");
        DumpMachine();

        // This changes the second qubit into state |+⟩ = (1/sqrt(2))(|0⟩ + |1⟩).
        H(qs[1]);
        Message("The system in now in state  (1/sqrt(2))(|10⟩ + |11⟩):");
        DumpMachine();

        // This changes the first qubit into state |-⟩ = (1/sqrt(2))(|0⟩ - |1⟩)
        H(qs[0]);
        Message("The system in now in state 0.5(|00⟩ + |01⟩ - |10⟩ - |11⟩):");
        DumpMachine();

        // The next lines entangle the qubits (don't worry about what exactly they do for now).
        H(qs[1]);
        CNOT(qs[0], qs[1]);
        Message("The system in now in entangled state (1/sqrt(2))(|00⟩ - |11⟩):");
        DumpMachine();

        // This returns the system into state |00⟩.
        ResetAll(qs);
    }
}
```

> 您可能已经注意到，每个演示都在代码末尾“重置”量子位，即将它们返回到 $\ket{0}$ 状态。 Q# 要求您将量子位返回到 $\ket{0}$ 状态，然后再在其范围结束时释放它们。
> 其原因是纠缠。
>
> 考虑在量子计算机上运行一个程序：量子位的数量非常有限，并且您希望在程序的其他部分重用释放的量子位。
如果那时它们尚未处于零状态，则它们可能仍然与尚未释放的量子位纠缠在一起，因此对它们执行的操作可能会影响程序其他部分的状态，从而导致错误且难以调试的行为。
>
> 当量子位超出其分配的块范围时，自动将其重置为零状态也是危险的：如果它们与其他量子位纠缠在一起，测量它们以重置它们可能会影响未释放的量子位的状态，从而改变程序的结果 - 开发人员不会注意到这一点。
>
> 量子位在释放之前应处于零状态的要求旨在提醒开发人员仔细检查所有必要的信息是否已从量子位中正确提取，并且它们不再与未释放的量子位纠缠在一起。
