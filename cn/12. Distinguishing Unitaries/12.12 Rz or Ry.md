# Rz 还是 Ry？

**输入：**

1. 角度 $\theta \in [0.01 \pi; 0.99 \pi]$。
2. 实现单量子位酉变换的操作：
[$R_z(\theta)$ 门](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.intrinsic/rz) 或 [$R_y(\theta)$ 门](https://learn.microsoft.com/qsharp/api/qsharp-lang/microsoft.quantum.intrinsic/ry)。

> 提醒一下，
> 
> $$R_z(\theta) = \begin{bmatrix} e^{-i\theta/2} & 0 \\ 0 & e^{i\theta/2} \end{bmatrix}$$
> $$R_y(\theta) = \begin{bmatrix} \cos\frac{\theta}{2} & -\sin\frac{\theta}{2} \\ \sin\frac{\theta}{2} & \cos\frac{\theta}{2} \end{bmatrix}$$

**输出：** 如果给定操作是 $R_z$ 门，则为 0；如果给定操作是 $R_y$ 门，则为 1。

该操作将定义伴随变量和受控变量。
您可以应用给定操作及其伴随/受控变体**任意次数**。

## 解决方案注释

这里的关键观察是 $R_z$ 是对角矩阵而 $R_y$ 不是，因此当应用于 $\ket{0}$ 状态时，前者将保持其不变（具有不可观察的额外相位），而后者将其转换为叠加 $\cos\frac{\theta}{2} \ket{0} + \sin\frac{\theta}{2} \ket{1}$。问题是，如果这两个状态不正交（​​并且对于 $\theta$ 的大多数值来说它们不会正交），如何区分它们？

任务描述给了你一个很大的提示：它允许你无限次地使用给定的酉，这指向概率解决方案（与此 kata 中所有先前问题中的确定性解决方案相反）。将酉应用于$\ket{0}$状态并测量结果；如果是$\ket{1}$，则酉必须是$R_y$，否则可以再次重复实验。经过几次迭代测量 $\ket{0}$ 后，您可以得出结论，单一元很可能是 $R_z$。

为了减少做出决定后的迭代次数，您可以多次应用酉，使整体旋转角度更接近 $\pi$：在 $R_y$ 门的情况下，这将允许您将状态旋转到更接近 $\ket{1}$ 状态，以便您以更高的概率检测到它。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Convert.*;
    import Std.Math.*;

    function ComputeRepetitions(angle : Double, offset : Int, accuracy : Double) : Int {
        mutable pifactor = 0;
        while (true) {
            let pimultiple = PI() * IntAsDouble(2 * pifactor + offset);
            let times = Round(pimultiple / angle);
            if AbsD(pimultiple - (IntAsDouble(times) * angle)) / PI() < accuracy {
                return times;
            }
            set pifactor += 1;
        }
        return 0;
    }

    operation DistinguishRzFromRy(theta : Double, unitary : (Qubit => Unit is Adj + Ctl)) : Int {
        use q = Qubit();
        let times = ComputeRepetitions(theta, 1, 0.1);
        mutable attempt = 1;
        mutable measuredOne = false;
        repeat {
            for _ in 1..times {
                unitary(q);
            }
            // for Rz, we'll never venture away from |0⟩ state, so as soon as we got |1⟩ we know it's not Rz
            if MResetZ(q) == One {
                set measuredOne = true;
            }
            // if we try several times and still only get |0⟩s, chances are that it is Rz
        } until (attempt == 4 or measuredOne)
        fixup {
            set attempt += 1;
        }
        return measuredOne ? 1 | 0;
    }
}
```
