# I⊗X 还是 CNOT？

**输入：** 实现两个量子位酉变换的操作：
$I \otimes X$（应用于第二个量子位的 $X$ 门）或 $CNOT$ 门，以第一个量子位为控制，以第二个量子位为目标。

**输出：** 如果给定操作是 $I \otimes X$，则为 0；如果给定操作是 $CNOT$ 门，则为 1。

该操作将接受一组量子位作为输入，但如果该数组为空或具有一个或两个以上量子位，则该操作将会失败。
该操作将定义伴随变量和受控变量。
您可以精确**一次**应用给定操作及其伴随/受控变体。

## 解决方案注释

让我们考虑一下这些门对基础状态的影响：

我们可以看到，将这两个门应用于第一个量子位处于 $\ket{1}$ 状态的状态会产生相同的结果，但将它们应用于第一个量子位处于 $\ket{0}$ 状态的状态会产生与第二个 qubuit 不同的状态。
这是有道理的，因为 $CNOT$ 门被定义为“如果控制量子位处于 $\ket{1}$ 状态，则将 $X$ 门应用于目标量子位，如果控制量子位处于 $\ket{0}$ 状态，则不执行任何操作”。

因此，最简单的解决方案是：分配两个处于 $\ket{00}$ 状态的量子位并对它们应用酉，然后测量第二个量子位；如果是`One`，则门是$I \otimes X$，否则是$CNOT$。

来源：Solution.qs

```qsharp
namespace Kata {
    operation DistinguishIXfromCNOT (unitary : (Qubit[] => Unit is Adj + Ctl)) : Int {
        use qs = Qubit[2];
        unitary(qs);
        return MResetZ(qs[1]) == One ? 0 | 1;
    }
}
```
