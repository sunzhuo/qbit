# Q# 量子位数据类型

在 Q# 中，量子位由 `Qubit` 数据类型表示。在物理量子计算机上，不可能直接访问量子位的状态，无论是读取其确切状态还是将其设置为所需状态，这种数据类型反映了这一点。相反，您可以使用量子门更改量子位的状态，并使用测量提取有关系统状态的信息。

话虽这么说，当您在量子模拟器而不是物理量子计算机上运行 Q# 代码时，您可以使用诊断功能来查看量子系统的状态。这对于学习和调试小型 Q# 程序都非常有用。

量子位不是普通的数据类型，因此这种类型的变量必须以稍微不同的方式声明和初始化（“分配”）。 `use` 语句分配一个（或多个）量子位，该量子位可以在使用该语句的范围结束之前使用： `use q = Qubit();` 分配一个量子位并将其绑定到变量 `q`。

新分配的量子位从状态 $\ket{0}$ 开始，并且在释放时必须返回到该状态。如果您尝试在除 $\ket{0}$ 之外的任何状态下释放量子位，将会导致运行时错误。当您稍后查看多量子位系统时，您会明白为什么它很重要。

## 量子态可视化

在继续之前，让我们学习一些可视化量子位的量子态的技术。

### 显示单量子位程序的量子状态

让我们从一个简单的场景开始：作用于单个量子位的程序。
该程序使用的量子系统的状态可以表示为长度为 2 的复向量，或者使用狄拉克符号，

$$\begin{bmatrix} \alpha \\ \beta \end{bmatrix} = \alpha\ket{0} + \beta\ket{1}$$

如果该程序在物理量子系统上运行，则无法从单个观察中获取有关程序执行的某个点的 $\alpha$ 和 $\beta$ 值的信息。
到目前为止，您需要重复运行该程序，对系统执行测量，然后汇总多次测量的结果以估计 $\alpha$ 和 $\beta$。

然而，在量子程序开发的早期阶段，程序通常在模拟器上运行——这是一种经典程序，它模拟小型量子系统的行为，同时拥有有关其内部状态的完整信息。
您可以利用这一点来做一些非物理的事情，例如窥视量子系统的内部以观察其确切状态而不干扰它！

`Std.Diagnostics` 命名空间中的 `DumpMachine` 函数可以让您做到这一点。 `DumpMachine` 的输出在全局相位范围内都是准确的，请记住全局相位没有任何物理意义。使用 `DumpMachine` 时，您可能会看到与您期望的状态相比，所有概率幅度都乘以某个复数。

### 演示：用于单量子位系统的 DumpMachine

以下演示展示了如何在 Q# 中分配量子位并检查其状态。您将使用 `DumpMachine` 在程序中的任何点输出系统状态，而不影响状态。

> 请注意，Q# 代码无法访问 `DumpMachine` 的输出，因此您无法在 Q# 中编写任何非物理代码！

来源：SingleQubitDumpMachineDemo.qs

```qsharp
namespace Kata {
    import Std.Diagnostics.*;

    @EntryPoint()
    operation RunExample() : Unit {
        // This line allocates a qubit in state |0⟩.
        use q = Qubit();
        Message("State |0⟩:");

        // This line prints out the state of the quantum computer.
        // Since only one qubit is allocated, only its state is printed.
        DumpMachine();

        // This line changes the qubit from state |0⟩ to state |1⟩.
        X(q);

        Message("State |1⟩:");
        DumpMachine();

        // This line changes the qubit to state |-⟩ = (1/sqrt(2))(|0⟩ - |1⟩).
        // That is, this puts the qubit into a superposition where the absolute
        // value of the probability amplitudes is 1/sqrt(2), which is
        // approximately 0.707107.
        H(q);

        Message("State |-⟩:");
        DumpMachine();

        // This line changes the qubit to state |-i⟩ = (1/sqrt(2))(|0⟩ - i|1⟩).
        S(q);

        Message("State |-i⟩:");
        DumpMachine();

        // This will put the qubit into an uneven superposition, where the
        // amplitudes of |0⟩ and |1⟩ have different absolute values.
        Rx(2.0, q);
        Ry(1.0, q);

        Message("Uneven superposition state:");
        DumpMachine();

        // This line returns the qubit to state |0⟩, which must be done before
        // the qubit is released or otherwise a runtime error might occur.
        Reset(q);
    }
}
```

`RunExample` 操作的确切行为取决于您使用的量子模拟器或处理器。

在这些演示中使用的模拟器上，此函数打印有关具有非零幅度的每个基本状态的信息，每行一个基本状态。
这包括有关状态幅度、测量该状态的概率以及状态相位的信息。

请注意，每一行都具有以下格式：

<table>
    <thead>
        <tr>
            <th>基础状态</th>
            <th>振幅</th>
            <th>测量概率</th>
            <th>阶段</th>
        </tr>
    </thead>
</table>

例如，状态 $\ket{0}$ 将表示如下：

<table>
    <tbody>
        <tr>
            <td>|0⟩</td>
            <td>1.0000+0.0000𝑖</td>
            <td>100.0000%</td>
            <td>↑ 0.0000</td></tr>
    </tbody>
</table>

> 需要注意的是，虽然我们从量子系统的状态角度讨论量子系统，但 Q# 在语言中没有任何量子态的表示。相反，状态是量子系统的内部属性，可以使用门进行修改。有关详细信息，请参阅有关量子态的 <a href="https://learn.microsoft.com/azure/quantum/concepts-dirac-notation#q-gate-sequences-equivalent-to-quantum-states" target="_blank">Q# 文档 </a>。
