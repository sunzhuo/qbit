# 使用部分测量进行状态准备

**输入：** 处于状态 $\ket{00}$ 的两个量子位。

**目标：** 使用后选择将量子位修改为状态 $\frac{1}{\sqrt{3}} \big(\ket{00} + \ket{01} + \ket{10}\big)$。

<br/>
<details>
  <summary><b>需要提示吗？</b></summary>
  考虑 3 量子位状态 $\frac{1}{2}(\ket{00} + \ket{01} + \ket{11}) \otimes \ket{0} + \frac{1}{2} \ket{11} \otimes \ket{1}$。
  当你测量第三个量子位时会发生什么？
</details>

## 解决方案注释

虽然可以直接使用单一旋转来准备状态 $\ket \psi$，但使用后选择来准备它更简单。

最初，您通过将 $H$ 门应用于前两个量子位，在前两个量子位上准备所有基本状态的相等叠加，并在 $\ket{0}$ 状态中分配一个额外的量子位：
$$\frac{1}{2} \big(\ket{00} + \ket{01} + \ket{10} + \ket{11}\big) \otimes \ket 0$$

前两个量子位的状态是您要准备的状态和您要丢弃的 $\ket{11}$ 状态的叠加。

现在，您想要将前三个基本状态与最后一个基本状态分开，并将这种分离存储在额外的量子位中。
例如，您可以保留您想要保留的基础状态的额外量子位 $\ket{0}$ 的状态，并将其切换到您想要丢弃的基础状态的 $\ket{1}$。
$CCNOT$ 门可以做到这一点，前两个量子位用作控制量子位，额外的量子位作为目标。
应用门时，如果两个控制量子位都处于 $\ket{11}$ 状态，则额外量子位的状态只会更改为 $\ket{1}$，这恰好标记了您要丢弃的状态：

$$CCNOT\frac{1}{2} \big(\ket{00\textbf{0}} + \ket{01\textbf{0}} + \ket{10\textbf{0}} + \ket{11\textbf{0}}\big) =
\frac{1}{2}\big(\ket{00} + \ket{01} + \ket{10} \big) \otimes \ket{\textbf{0}} + \frac{1}{2}\ket{11} \otimes \ket{\textbf{1}} $$

最后，您只测量额外的量子位；这会导致系统部分崩溃到测量结果定义的状态：
* 如果结果为 $\ket{0}$，则前两个量子位将崩溃为基础状态的线性组合状态，其中额外量子位处于状态 $\ket{0}$，也就是说，它们最终处于目标状态 $\frac{1}{\sqrt3}\big(\ket{00} + \ket{01} + \ket{10}\big)$。
* 如果结果是 $\ket{1}$，则前两个量子位会崩溃到状态 $\ket{11}$，因此您的目标未实现。好消息是，这种情况只发生在 25% 的情况下，您只需将我们的量子位重置为 $\ket{00}$ 状态，然后重试。

> Q# 有一个内置的 <a href="https://learn.microsoft.com/azure/quantum/user-guide/language/expressions/conditionalloops#repeat-expression" target="_blank">repeat-until-success (RUS) 循环 </a>，在这种情况下会派上用场。
> * 主要操作（应用 $H$ 和 $CCNOT$ 门和测量）在循环的 `repeat` 部分中描述，该部分指定其主体。  
> * `until` 部分指定将打破循环的条件。在这种情况下，测量结果需要为 `Zero` 才能表明您成功。  
> * 最后，如果不满足成功标准，`fixup` 部分允许您在重试之前清理循环体执行的结果。在这种情况下，您将前两个量子位重置回 $\ket{00}$ 状态。

来源：Solution.qs

```qsharp
namespace Kata {
    operation PostSelection(qs : Qubit[]) : Unit {
        use anc = Qubit();
        mutable res = Zero;
        repeat {
            ApplyToEach(H, qs);
            Controlled X(qs, anc);
            set res = MResetZ(anc);
        }
        until (res == Zero)
        fixup {
            ResetAll(qs);
        }
    }
}
```
