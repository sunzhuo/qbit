# 重建消息（查理的任务）

Alice 有一个状态为 $\ket{\psi}$ 的消息量子位需要传送。她以与 `SendMessage` 练习中相同的方式将其与来自 $\ket{\Psi^{3}}$ 的她自己的量子位纠缠在一起，并提取了两个经典位，以便将它们发送给 Charlie。 Bob 还从 $\ket{\Psi^{3}}$ 测量了他自己的量子位，并将结果发送给 Charlie。使用从 Alice 收到的两个经典位和从 Bob 收到的一个经典位，将 Charlie 的量子位转换为所需的状态。

**输入：** 
1. Charlie 是纠缠量子位三人组 `qCharlie` 的一部分。
2. 从 Alice 收到的经典位元组，采用 `SendMessage` 练习中使用的格式。
3. 由 Bob 的量子位测量产生的经典位。

**目标：** 
将 Charlie 的量子位 `qCharlie` 转换为消息量子位最初的状态。

## 解决方案注释

与传统的隐形传输协议相同，Charlie 的量子位现在包含有关隐形传输状态的幅度 $\alpha$ 和 $\beta$ 的信息，但它需要根据从 Alice 和 Bob 收到的经典消息进行校正，以精确匹配隐形传输状态。由于存在三个布尔值，因此总共有八种可能性。
- 对于 000，无需更改。
- 对于001，仅需要X 校正。
- 对于010，仅需要X 校正。
- 对于 011，不需要更改或结果操作应该是 Identity。
- 对于 100，仅需要 Z 校正。
- 对于 101，需要 Z 和 X 校正。
- 对于 110，需要 Z 和 X 校正。
- 对于111，仅需要Z 校正。

综上所述，如果`qMessage`的测量结果为1，我们需要应用$Z$门，如果`qAlice`和`qBob`的测量结果不同，我们需要应用$X$门。

来源：Solution.qs

```qsharp
namespace Kata {
    operation ReconstructMessageWhenThreeEntangledQubits(qCharlie : Qubit, (b1 : Bool, b2 : Bool), b3 : Bool) : Unit {
        if b1 {
            Z(qCharlie);
        }
        if b2 != b3 {
            X(qCharlie);
        }
    }
}
```
