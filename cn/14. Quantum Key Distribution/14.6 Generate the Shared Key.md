# 生成共享密钥

现在，Alice 拥有她发送的位值的列表以及她准备每个量子位的基础，而 Bob 拥有他用于测量每个量子位的基础列表和测量结果列表。为了提取共享密钥，他们需要弄清楚何时使用相同的基础，保留这些步骤中的数据，并丢弃使用不同基础的步骤中的数据。

**输入：**

1. `basesAlice` 和 `basesBob`：`Bool` 长度为 $N$ 的数组，分别描述 Alice 和 Bob 对编码/测量基数的选择。
2. `bits`：长度为$N$的`Bool`数组，描述Alice发送的位或Bob的测量结果。

**输出**：表示协议生成的共享密钥的 `Bool` 数组。

> 您不需要同时知道 Alice 和 Bob 的位来计算共享密钥 - 只需一组位就足够了！

## 解决方案注释

如果爱丽丝和鲍勃交换一个量子位并使用相同的基础来准备和测量它，那么鲍勃测量产生的位将与爱丽丝编码的位相同。因此，他们不需要共享他们通过经典通信渠道选择或获得的比特。共享每个量子位所使用的基数足以了解它们的位是否匹配。

要完成此任务，您需要执行以下步骤：

1.声明一个空的可变数组，我们将其命名为`key`。
2. 根据 Alice 和 Bob 使用的基数之间的比较，决定可以添加到密钥中的位。您可以使用 $0$ 到 $N - 1$ 范围内的索引进行迭代，并比较相应位置的碱基。
3. 返回所需的`key`。

来源：Solution.qs

```qsharp
namespace Kata {
    function GenerateSharedKey(basesAlice : Bool[], basesBob : Bool[], bits : Bool[]) : Bool[] {
        mutable key = [];
        for i in 0 .. Length(bits) - 1 {
            if basesAlice[i] == basesBob[i] {
                set key += [bits[i]];
            }
        }

        return key;
    }
}
```
