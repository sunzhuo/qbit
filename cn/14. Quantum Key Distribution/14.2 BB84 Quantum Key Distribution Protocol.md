# BB84 量子密钥分发协议

BB84 协议以其发明者 Charles H. Bennett 和 Gilles Brassard 的名字命名，并以其出版年份命名，是最早提出的量子密钥分发协议之一，也可能是最著名的协议。

BB84 协议由两个主要阶段组成：

1. 第一阶段，Alice按照一定的程序准备好各个量子比特，然后通过量子通道发送给Bob进行测量。 Alice 记录了她在准备量子位时所做的经典决策，Bob 记录了他在测量量子位和测量结果时所做的决策。

2. 第二阶段完全是经典的后处理和通信：Alice 和 Bob 讨论第一阶段的数据并提取一个经典的随机位串，他们可以将其用作共享密钥。

让我们首先看看爱丽丝如何准备她的量子位以将其发送给鲍勃。

Alice 对于每个量子位有两种选择：准备它的基础以及她想要编码的位值。
在第一个基础（计算基础）中，Alice 准备状态 $\ket{0}$ 和 $\ket{1}$，其中 $\ket{0}$ 表示密钥位值 `0`，$\ket{1}$ 表示密钥位值 `1`。
第二个基，Hadamard 基（有时也称为对角基），使用状态 $\ket{+} = \frac{1}{\sqrt2}(\ket{0} + \ket{1})$ 表示密钥位值 `0`，使用 $\ket{-} = \frac{1}{\sqrt2}(\ket{0} - \ket{1})$ 表示密钥位值 `1`。

协议中使用的基数是经过选择的，如果窃听者试图测量传输中的量子位并选择了错误的基数，那么他们只会以相同的概率得到 0 或 1 的测量结果。

Alice 必须为她准备的每个量子位做出两个随机选择，一个用于准备的基础，另一个用于她想要发送的位值，以 $50\%$ 概率选择每个选项。
如果 Alice 决定发送 $N$ 量子位，她需要做出 $2N$ 随机选择，通常实现为两个 $N$ 选择数组。

一旦鲍勃从爱丽丝那里收到量子位，他需要决定在哪个基础上（计算基础或哈达玛基础）来测量它们中的每一个，并且这些决定也是随机的，每个基础都以 $50\%$ 概率选择。

最后，在协议第一阶段结束时，Alice 拥有她发送的位值的列表以及她准备每个量子位的基础，而 Bob 拥有他用于测量每个量子位的基础列表。

为了提取共享密钥，他们需要弄清楚它们何时使用相同的基数，并从使用不同基数的量子位中丢弃数据。如果 Alice 和 Bob 没有使用相同的基础来准备和测量量子位，Bob 得到的测量结果将只是随机位，0 和 1 结果的概率均为 $50\%$。但如果他们使用相同的基础，鲍勃的测量结果将与爱丽丝发送的比特相匹配。

这意味着，通过通过公共经典通信通道交换有关 Alice 和 Bob 用于准备和测量的基数的信息，他们可以推断出他们保密的位列表中的哪些部分是相同的，并将它们用作共享密钥！
