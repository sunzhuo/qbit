# 一顶点弱着色有效吗？ （量子）

**输入：**

1. 图 $V$ ($V \leq 6$) 中的顶点数。
2. $E$ 整数元组数组，表示图的边 ($E \leq 12$)。
每个元组给出边的起始和结束顶点的索引。
顶点编号为 $0$ 到 $V - 1$。
3. 处于任意状态 $\ket{x}$ 的 $2V$ 量子位数组，表示使用四种颜色 ($nBits = 2$) 的顶点指定着色，其格式与练习“从量子位数组读取着色”中的格式相同。
4. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。
5. 图中 $0$ 和 $V - 1$（含）之间的顶点索引。

**目标：**
实现一个量子预言机，检查给定顶点在给定图的给定着色中是否为弱着色，
也就是说，它是孤立的还是连接到至少一个不同颜色的顶点。
    
让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

要实现此检查，您需要首先计算连接到给定顶点的顶点数 
并分配一组辅助量子位，每个相邻顶点一个。如果相应边连接的顶点具有相同的颜色，这些量子位的状态将会翻转。 
使用先前任务中的操作 `Oracle_ColorEquality` 可以轻松完成此操作，类似于验证图形的顶点着色时的操作方式。

现在，为了使顶点的着色有效，它需要没有邻居，或者至少有一个不同颜色的邻居，因此辅助量子位数组应该处于除 $\ket{1...1}$ 之外的任何状态。为了实现这一点，您可以使用 $X$ 门无条件翻转目标量子位 $\ket{y}$，然后仅当量子位处于状态 $\ket{1...1}$ 时再次翻转它。

确保在评估最终结果后取消计算对辅助量子位的更改，以便在发布之前保持它们干净。

```qsharp
namespace Kata {
    import Std.Arrays.*;

    operation Oracle_WeakColoring_OneVertex(
        V : Int,
        edges : (Int, Int)[],
        x : Qubit[],
        y : Qubit,
        vertex : Int
    ) : Unit is Adj + Ctl {
        let neighborEdges = Filtered((a, b) -> a == vertex or b == vertex, edges);
        let nNeighbors = Length(neighborEdges);
        use sameColorChecks = Qubit[nNeighbors];
        within {
            for ((a, b), checkQubit) in Zipped(neighborEdges, sameColorChecks) {
                Oracle_ColorEquality(
                    x[a * 2 .. a * 2 + 1],
                    x[b * 2 .. b * 2 + 1],
                    checkQubit
                );
            }
        } apply {
            X(y);
            if nNeighbors > 0 {
                Controlled X(sameColorChecks, y);
            }
        }
    }

    operation Oracle_ColorEquality(x0 : Qubit[], x1 : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        within {
            for i in 0..Length(x0) - 1 {
                CNOT(x0[i], x1[i]);
            }
        } apply {
            ApplyControlledOnInt(0, X, x1, y);
        }
    }
}
```
