# 颜色相等吗？

**输入：**

1. 处于任意状态 $\ket{x_0}$ 的 $nBits$ 量子位阵列，表示第一种颜色。
2. 处于任意状态 $\ket{x_1}$ 的 $nBits$ 量子位阵列，表示第二种颜色。
3. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：**
实现一个量子预言机，检查两个给定的颜色是否相等。
换句话说，如果 $x_0 = x_1$，则翻转量子位 $\ket{y}$ 的状态，否则保持不变。
    
让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

目标是当且仅当数组 `x0` 和 `x1` 中每个匹配的量子位对处于相同状态时，翻转量子位 $\ket{y}$。

您可以通过计算两个量子位的 $\textrm{XOR}$ 来检查它们是否处于相同状态：如果它们的状态相同，则它们的 $\textrm{XOR}$ 将是 $0$。您可以使用 $\textrm{CNOT}$ 门就地计算两个量子位的 $\textrm{XOR}$，例如，使用数组 `x0` 的量子位作为控制，使用数组 `x1` 的量子位作为目标。

使用 $nBits$ $\textrm{CNOT}$ 门来计算所有成对 $\textrm{XOR}$ 后，仅当 `x1` 中的所有量子位都处于 $\ket{0}$ 状态时，您才需要翻转目标量子位 $\ket{y}$。 
这可以通过使用零控制的 $X$ 门（即 `ApplyControlledOnInt(0, X, x1, y)`）来完成。

最后，您需要取消按位 $\textrm{XOR}$ 的计算，以确保 `x1` 中的量子位返回到其原始状态。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Oracle_ColorEquality(x0 : Qubit[], x1 : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        within {
            for i in 0..Length(x0) - 1 {
                CNOT(x0[i], x1[i]);
            }
        } apply {
            ApplyControlledOnInt(0, X, x1, y);
        }
    }
}
```
