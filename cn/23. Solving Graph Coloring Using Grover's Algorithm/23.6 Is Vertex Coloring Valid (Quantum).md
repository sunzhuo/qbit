# 顶点着色有效吗？ （量子）

**输入：**

1. 图 $V$ ($V \leq 6$) 中的顶点数。
2. $E$ 整数元组数组，表示图的边 ($E \leq 12$)。
每个元组给出边的起始和结束顶点的索引。
顶点编号为 $0$ 到 $V - 1$。
3. 处于任意状态 $\ket{x}$ 的 $2V$ 量子位数组，表示使用四种颜色 ($nBits = 2$) 的顶点指定着色，其格式与练习“从量子位数组读取着色”中的格式相同。
4. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

例如，图`0 -- 1 -- 2`具有$V = 3$和`edges = [(0, 1), (1, 2)]`。 
该图的颜色分配用 $6$ 量子位表示。
一些有效的着色是 $\ket{000110}$（顶点 `0, 1, 2` 分配颜色 `0, 1, 2`）和 $\ket{011001}$（顶点 `0, 1, 2` 分配颜色 `1, 2, 1`）。

**目标：**
实现一个量子预言机，检查该图的给定着色是否是有效的顶点着色，
也就是说，分配给与一条边相连的每对顶点的颜色是否不同。
    
让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

您需要分配一组辅助量子位，一个对应于图的每个边，如果相应边连接的顶点具有相同的颜色，则它们的状态将翻转。 
使用上一个任务中的操作 `Oracle_ColorEquality` 可以轻松完成此操作。

然后，您需要检查这些辅助量子位是否仍处于状态$\ket{0...0}$；如果是，则所有必要的颜色对都是不同的，着色是有效的。

由于着色是以量子位数组的形式提供的，每个顶点有两个量子位（2 个量子位 = 4 个基本状态 = 4 种颜色），因此您必须采用正确的着色块来提取每个顶点的颜色。您可以推断出顶点 $j$ 的颜色是在位置 $2j$ 和 $2j+1$ 的量子位中编码的。

确保在评估最终结果后取消计算对辅助量子位的更改，以便在发布之前保持它们干净。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Oracle_VertexColoring(V : Int, edges: (Int, Int)[], x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        let edgesNumber = Length(edges);
        use conflicts = Qubit[edgesNumber];
        within {
            for i in 0 .. edgesNumber - 1 {
                let (v0, v1) = edges[i];
                Oracle_ColorEquality(x[2 * v0 .. 2 * v0 + 1], 
                                     x[2 * v1 .. 2 * v1 + 1], conflicts[i]);
            }
        } apply {
            ApplyControlledOnInt(0, X, conflicts, y);
        }
    }

    operation Oracle_ColorEquality(x0 : Qubit[], x1 : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        within {
            for i in 0..Length(x0) - 1 {
                CNOT(x0[i], x1[i]);
            }
        } apply {
            ApplyControlledOnInt(0, X, x1, y);
        }
    }
}
```
