#Q 中的保利·盖茨#

以下示例包含演示如何在 Q# 中应用门的代码。它设置了一系列量子态，然后显示将 $X$ 门应用于每个量子态的结果。

前面的 kata 提到 Q# 中的量子位状态不能直接分配或访问。相同的逻辑扩展到量子门：将门应用于量子位会修改该量子位的内部状态，但不会返回量子位的结果状态。这就是为什么您从不将这些门的输出分配给此演示中的任何变量的原因 - 它们不产生任何输出。

同样的原理也适用于将多个门连续应用于一个量子位。在数学符号中，将 $X$ 门和 $Z$ 门应用到状态 $\ket{\psi}$ 表示为 $Z(X(\ket{\psi}))$，因为将门应用到一个状态的结果是另一个状态。在 Q# 中，应用门不会返回任何内容，因此您不能使用其输出作为另一个门的输入 - 像 `Z(X(q))` 这样的东西不会产生预期的结果。相反，要将多个门应用于同一个量子位，您需要按照它们的应用顺序分别调用它们：

```qsharp
X(q);
Z(q);
```

此 kata 中涵盖的所有基本门都是 Intrinsic 命名空间的一部分。此外，Diagnostics 命名空间中的函数 `DumpMachine` 用于打印量子模拟器的状态。

资料来源：PauliGates.qs

```qsharp
namespace Demo {
    // To use elements from a namespace, you need to use the `import` keyword to
    // access them.
    import Std.Diagnostics.*;

    @EntryPoint()
    operation PauliGatesUsage() : Unit {
        // This allocates a qubit for us to work with.
        use q = Qubit();

        // This will put the qubit into an uneven superposition |𝜓❭, where the
        // amplitudes of |0⟩ and |1⟩ have different absolute values.
        Ry(1.0, q);

        Message("Qubit in state |𝜓❭:");
        DumpMachine();

        // Let's apply the X gate; notice how it swaps the amplitudes of the
        // |0❭ and |1❭ basis states.
        X(q);
        Message("Qubit in state X|𝜓❭:");
        DumpMachine();

        // Applying the Z gate adds -1 relative phase to the |1❭ basis states.
        Z(q);
        Message("Qubit in state ZX|𝜓❭:");
        DumpMachine();

        // Finally, applying the Y gate returns the qubit to its original state
        // |𝜓❭, with an extra global phase of i.
        Y(q);
        Message("Qubit in state YZX|𝜓❭:");
        DumpMachine();

        // This returns the qubit to state |0❭.
        Reset(q);
    }
}
```
