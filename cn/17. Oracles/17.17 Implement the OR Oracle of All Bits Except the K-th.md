# 实现除第 K 位之外的所有位的 OR Oracle

**输入：**

1. $N \geq 2$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
  2. 整数 $k$ 使得 $0 \leq k < N$。

**目标：**

如果输入寄存器中 $x$（不考虑第 $k$ 位）的任何位为 $1$，则翻转基本状态 $\ket{x}$ 的符号。换句话说，排除第$k$个量子位的输入寄存器不应该处于全零状态以翻转输入寄存器的符号。第 $k$ 个量子位的状态不会影响结果。

*请随意探索使用或不使用辅助量子位来实现此操作。*

**示例：**

* 如果查询寄存器处于状态$\ket{010}$和$k=0$，则翻转寄存器的符号。
* 如果查询寄存器处于状态$\ket{010}$和$k=1$，则不执行任何操作。

## 解决方案注释

解决此任务的最简单方法是基于之前练习中的标记 OR 预言的实现
并使用相位反冲技巧将其转换为相位神谕。

由于在此任务中您正在评估除第 $k$ 位之外的所有位的 OR 运算，因此您需要从列表中排除量子位 `x[k]` 
标记预言机的控制量子位。您可以使用切片来做到这一点：`x[...k - 1]` 获取第 $k$ 之前的量子位子数组 
1，`x[k + 1...]` 获取第 $k$ 之后的量子位子数组。您的控制量子位数组将是这些的串联 
两个数组。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Or_Oracle(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        X(y);
        ApplyControlledOnInt(0, X, x, y);
    }

    operation OrOfBitsExceptKth_Oracle(x : Qubit[], k : Int) : Unit is Adj + Ctl {
        use minus = Qubit();
        within {
            X(minus);
            H(minus);
        } apply {
            Or_Oracle(x[...k - 1] + x[k + 1...], minus);
        }
    }
}
```
