# 实现 OR Oracle

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
  2. 处于任意状态 $\ket{y}$ 的量子位（目标量子位）。

**目标：**

如果输入寄存器处于任何基本状态，则翻转 $\ket{y}$ 的状态
$\ket{00...0}$（全零状态）除外。

**示例：**

* 如果查询寄存器处于状态 $\ket{10000001}$、$\ket{11101101}$ 或 $\ket{0010101}$，则翻转状态 $\ket{y}$。
* 如果查询寄存器处于状态$\ket{000}$，则不执行任何操作。

<br/>
<details>
  <summary><b>在实现此预言之前，请回答以下问题：您是否正在实现标记预言机或相位预言机？</b></summary>
    这是一个标记预言，因为您根据输入 $\ket{x}$ 的状态翻转目标量子位 $\ket{y}$ 的状态。
</details>

<br/>
<details>
  <summary><b>需要提示吗？</b></summary>
  您需要为除 $\ket{00...0}$ 之外的每个输入翻转 $\ket{y}$ 的状态，或者无条件翻转它，然后翻转 $\ket{00...0}$ 状态。   您可能会发现 Q# 库函数 <code>ApplyControlledOnInt</code> 在您的实现中很有用。
</details>

## 解决方案注释

如果仅当输入寄存器处于 $\ket{0...0}$ 状态时才需要翻转目标量子位的状态，该怎么办？
在这种情况下，您可以将 `ApplyControlledOnInt` 与控制模式 $0$ 一起使用，该控制模式对应于 $\ket{0}$ 状态下的所有控制量子位。

在此问题中，您需要将 $\ket{0...0}$ 基础状态与所有其他状态分开，但会对目标量子位产生相反的影响：您需要针对所有其他输入状态翻转它，而不是针对此输入状态翻转它。
或者，您可以将其视为首先翻转所有状态的目标量子位的状态，然后仅针对该基本状态取消翻转（或再次翻转）。您可以通过在 `ApplyControlledOnInt` 之前或之后应用 $X$ 门来实现此目的。

请注意，您可以在计算期间修改输入寄存器的状态（这就是 `ApplyControlledOnInt` 函数在幕后所做的事情）。然而，除了最后的修改之外，有必要撤消这些修改（“取消计算”更改），以便预言机将保留输入（如果它是基本状态）。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Or_Oracle(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        X(y);
        ApplyControlledOnInt(0, X, x, y);
    }
}
```
