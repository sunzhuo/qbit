# 实施会议预言机

假设您想安排与同事 Jasmine 的会议。
你们每周工作五天，$\ket{x}$ 和 $\ket{jasmine}$ 是代表您和 Jasmine 日程安排的 5 位状态。
日程安排指示人员当天是否忙碌：$1$ 位表示人员当天忙碌，而 $0$ 表示人员当天有空参加会议。实现一个函数，确定您和 Jasmine 是否可以在一周内安排会议，即是否有一天两个日程安排同时具有 $0$。

**输入：**

1. 任意状态的 5 个量子位 $\ket{x}$ 代表您本周的日程安排（输入/查询寄存器）。
  2. 5 个处于任意状态的量子位 $\ket{jasmine}$ 代表 Jasmine 本周的日程安排（输入/查询寄存器）。
  3. 处于任意状态 $\ket{y}$ 的量子位（目标量子位）。

**目标：**

如果您和 Jasmine 在一周内至少有一天在同一天都有空，则翻转 $\ket{y}$ 的状态。  回想一下，$0$ 意味着一个人在那天有空。

**示例：**

* 如果 $\ket{x}=\ket{10101}$ 和 $\ket{jasmine}=\ket{01010}$，则不执行任何操作（没有哪一天你们俩都是空闲的）。
* 如果 $\ket{x}=\ket{10001}$ 和 $\ket{jasmine}=\ket{01010}$，则翻转状态 $\ket{y}$（周三你们都有空）。
* 如果 $\ket{x}=\ket{00000}$ 和 $\ket{jasmine}=\ket{00000}$，则翻转状态 $\ket{y}$（你们整周都有空）。
* 如果 $\ket{x}=\ket{11111}$ 和 $\ket{jasmine}=\ket{11111}$，则什么都不做（你们整个星期都很忙）。

## 解决方案注释

在此问题中，您需要分配额外的量子位（一周中的每一天一个）作为临时变量。
这些变量中的每一个都会跟踪当天两个人是否都有空，即两个输入量子位数组是否 
在相应位置有位 $0$。您可以在翻转后使用 $CCNOT$ 门计算每个变量 
使用 $X$ 门的两个量子位的状态。

之后，您可以重复使用标记 OR 预言机来检查是否有任何一天处于 $\ket{1}$ 状态。

当您在较大的计算期间分配量子位以供临时使用时，您需要确保它们返回到 $\ket{0}$ 
释放前的状态。不过，与您之前在演示中分配量子位的经验不同，这次您不能 
为此测量这些量子位 - 它们可能与您仍在使用的量子位纠缠在一起，因此测量将 
崩溃整个系统的状态，而不仅仅是辅助量子位。相反，你必须取消计算这些量子位的状态， 
类似于取消计算对输入量子位所做的任何更改，以便能够将受控门应用于正确的模式。
`within ... apply` 构造在这里也很有帮助。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Arrays.*;

    operation Or_Oracle(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        X(y);
        ApplyControlledOnInt(0, X, x, y);
    }

    operation Meeting_Oracle(x : Qubit[], jasmine : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        use qs = Qubit[Length(x)];
        within {
            for i in IndexRange(qs) {
                // flip q[i] if both x and jasmine are free on the given day
                X(x[i]);
                X(jasmine[i]);
                CCNOT(x[i], jasmine[i], qs[i]);
            }
        } apply {
            Or_Oracle(qs, y);
        }
    }
}
```
