# 实现相位预言机

**输入：**
  处于任意状态的 3 个量子位 $\ket{x}$（输入/查询寄存器）。

**目标：**

如果输入寄存器处于状态，则翻转输入状态 $\ket{x}$ 的符号
状态 $\ket{111}$（编码整数 $7$），否则保持输入寄存器不变。  
不要分配额外的量子位来执行此操作。

**示例：**

* 如果查询寄存器处于状态$\ket{111}$，则翻转其符号。
* 如果查询寄存器处于状态 $\ket{010}$ 或 $\ket{101}$，则不执行任何操作。

<details>
  <summary><b>需要提示吗？</b></summary>
  为了解决这个问题，你需要找到一个只会翻转 $\ket{111}$ 基态符号的门。  哪个单量子位门翻转基本状态 $\ket{1}$ 的符号，但不翻转 $\ket{0}$？如何修改这个门来解决这个问题？
</details>

## 解决方案注释

您可以使用受控 $Z$ 门翻转基本状态 $\ket{111}$ 的相位，其中任意两个量子位作为控制，第三个量子位作为目标。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Arrays.*;

    operation IsSeven_PhaseOracle(x : Qubit[]) : Unit is Adj + Ctl {
        Controlled Z(Most(x), Tail(x));
    }
}
```

考虑一下这个预言机如何在两个基础状态下发挥作用：
$$U_{7,phase} \ket{111} = -\ket{111}$$
$$U_{7,phase} \ket{110} = \ket{110}$$

您可以看到，如果输入是基础状态（除了添加全局阶段），$U_{7,phase}$ 不会更改输入，并且 $U_{7,phase}$ 不会更改状态的范数（$U_{7,phase}$ 是酉运算符）。

然而，如果你把这个预言应用到叠加状态，那会是什么样子呢？

假设 $\ket{\beta}$ 是状态 $\ket{110}$ 和 $\ket{111}$ 的相等叠加：
$$\ket{\beta} = \frac{1}{\sqrt{2}} \big(\ket{110} + \ket{111}\big) = \ket{11} \otimes \frac{1}{\sqrt{2}} \big(\ket{0} + \ket{1}\big) = \ket{11} \otimes \ket{+} = \ket{11+}$$

让我们考虑一下运算符 $U_{7,phase}$ 如何作用于此状态：

$$U_{7,phase} \ket{\beta} = U_{7,phase} \Big[\frac{1}{\sqrt{2}} \big(\ket{110} + \ket{111}\big)\Big] =$$

$$= \frac{1}{\sqrt{2}} \big(U_{7,phase} \ket{110} + U_{7,phase} \ket{111}\big) =$$

$$= \frac{1}{\sqrt{2}} \big(\ket{110} - \ket{111}\big) := \ket{\gamma}$$

在此操作期间您的输入状态是否被修改？让我们简化一下 $\ket{\gamma}$：

$$\ket{\gamma} = \frac{1}{\sqrt{2}} \big(\ket{110} - \ket{111}\big) =$$

$$= \ket{11} \otimes \frac{1}{\sqrt{2}} \big(\ket{0} - \ket{1}\big) =$$

$$= \ket{11} \otimes \ket{-} = \ket{11-} \neq \ket{\beta}$$

在这里您可以看到，如果输入状态是基础状态的“叠加”，则预言机会修改输入，因为相位预言机只会修改基础状态的符号。  因此，当提供叠加态作为预言机的输入时，可以通过量子预言机的应用来修改输入状态。

> 还值得注意的是，虽然预言机在提供叠加状态时修改了输入，但它并没有修改该状态的范数。  作为练习，您可以通过采用 $\ket{\beta}$ 和 $\ket{\gamma}$ 的范数自行验证这一点，这两者都会产生 $1$ 值。
>
> 作为另一个练习，考虑如何以编程方式区分输入和输出状态？  是否有一个操作可以应用于初始状态 $\ket{\beta}$ 和最终状态 $\ket{\gamma}$ 以表明这两个状态通过测量并不等效？  作为提示，请考虑如何将叠加状态 $\ket{\beta}$ 和 $\ket{\gamma}$ 转换为基础状态。
