# 交替位模式功能的阶段预言机

考虑函数 $f(x)$，它接受三位输入，如果 $x=101$ 或 $x=010$ 返回 $1$，否则返回 $0$。

实现此函数的相位预言机将三个量子位的数组作为输入，翻转基本状态 $\ket{101}$ 和 $\ket{010}$ 的符号，并保持其余基本状态不变。让我们看看这个预言对叠加状态的影响。

来源：PhaseOracleAltBit.qs

```qsharp
namespace Kata {
    import Std.Diagnostics.*;

    // This operation implements the oracle; we will learn how to implement oracles later in the kata
    operation AlternatingBitPattern_PhaseOracle(x : Qubit[]) : Unit is Adj + Ctl {
        use q = Qubit();
        X(q);
        ApplyControlledOnBitString([false, true, false], Z, x, q);
        ApplyControlledOnBitString([true, false, true], Z, x, q);
        X(q);
    }

    @EntryPoint()
    operation PhaseOracle_Demo() : Unit {
        use q = Qubit[3];
        ApplyToEachA(H, q);

        Message("Starting state (equal superposition of all basis states):");
        DumpMachine();

        // Apply the oracle
        AlternatingBitPattern_PhaseOracle(q);

        // Print the resulting state; notice which phases changed
        Message("State after applying the phase oracle:");
        DumpMachine();

        ResetAll(q);
    }
}
```

多量子位门 kata 在讨论受控门时介绍了 Q# 标准库提供的 `ApplyControlledOnBitString` 函数。
`ApplyControlledOnBitString` 函数定义了由位掩码指定的状态控制的门的变体。例如，位掩码 `[true, false]` 意味着仅当两个控制量子位处于 $\ket{10}$ 状态时才应应用门。

实现此变体的步骤顺序是：

1. 将 $X$ 门应用于与位掩码的 `false` 元素相对应的每个控制量子位。此后，如果控制量子位以 $\ket{10}$ 状态开始，它们最终将处于 $\ket{11}$ 状态，如果它们以任何其他状态开始，它们最终将处于除 $\ket{11}$ 之外的任何状态。
2.应用常规控制版闸门。
3. 将 $X$ 门应用于相同的量子位，将它们返回到原始状态。

> 请注意，上面演示中的输入状态是所有基本状态的相等叠加。
应用预言后，所有幅度的绝对值都相同，但状态 $\ket{010}$ 和 $\ket{101}$ 的相位翻转为负。
> 回想一下，这两个状态正是 $f(x) = 1$ 的输入，因此它们正是我们期望经历相位翻转的两个状态！
