# QPE 算法：实际考虑

现在您已经学会了实现 QPE 算法，让我们讨论一下其应用的几个实际方面。

### 将 QPE 应用于特征向量的叠加

首先，算法输入包括要估计其特征相位的酉特征向量。在某些应用中，特征向量要么是未知的，要么是难以准确准备。这些场景有解决方法吗？

为了回答这个问题，让我们看看如果您对不是酉 $U$ 本征态的状态 $\ket{\phi}$ 运行相位估计算法，会发生什么。

对于任何酉 $U$，您可以找到由其特征向量组成的基。这意味着任何状态 $\ket{\phi}$ 都可以表示为特征向量 $\ket{\psi_k}$ 的叠加：
$$\ket{\phi} = \sum_k a_k\ket{\psi_k}$$

您可以注意到，QPE 算法的所有步骤直到最终测量都是线性变换。这意味着您可以将它们独立地应用于叠加中的每一项。

最终测量之前的系统状态如下所示：

$$\sum_k a_k \ket{j_k} \ket{\psi_k}$$

这里 $j_k$ 是对应于特征向量 $\ket{\psi_k}$ 的特征相位的 $n$ 二进制数字的整数表示。

现在，当您测量第一个寄存器时，此测量将导致叠加的崩溃：您将获得描述一个特征相的值 $j_k$ 之一，第二个寄存器将崩溃到匹配的特征向量。

这意味着，如果您正在寻找酉的*任何*本征相，您可以使用任何状态代替本征向量来运行 QPE 算法。如果您知道要使用的特征向量，但精确准备很困难，则可以近似准备它并在此状态上运行 QPE 算法，以高概率获得与其相关的特征相位。

### 不是 $n$ 位二进制分数的特征相

上一课中的算法推导假设特征相位 $\theta$ 可以精确地以 $n$ 二进制位精度写下来。然而，情况并非总是如此。如果无法使用精确的 $n$ 二进制位写出给定特征向量的特征相位，会发生什么情况？

事实证明，以 $n$ 位精度运行的 QPE 算法仅对于可以精确表示为 $n$ 位二进制分数的特征相位具有确定性。对任何没有这种表示的特征相运行它会产生概率结果。与真实本征相位最接近的 $n$ 位分数的结果以最高概率产生，但也有可能获得与真实本征相位相距甚远的结果。

通过以下演示，您可以针对不同特征相使用 QPE 算法，并观察其在不同场景中的行为。
例如，您可以看到具有特征相位 $\frac1{16} = 0.0625$ 的门 `R1Frac(1, 3, _)` 的运行相位估计
$n = 3$ 位精度会产生不确定的结果，两个最常见的结果是 $0$ 和 $\frac18 = 0.125$。

来源：QuantumPhaseEstimationDemo.qs

```qsharp
namespace Kata {
    import Std.Convert.*;
    import Std.Diagnostics.*;
    import Std.Math.*;

    @EntryPoint()
    operation QuantumPhaseEstimationDemo() : Unit {
        // Experiment with the parameters to explore algorithm behavior for different eigenphases!
        // Use R1Frac(k, n, _) for eigenvalue exp(2π𝑖 k/2^(n+1)), eigenphase k/2^(n+1)
        // or R1(theta, _) for eigenvalue exp(2π𝑖 theta/2), eigenphase theta/2
        // Here are some convenient unitaries and their eigenphases
        // R1Frac(1, 0, _)   |  0.5
        // R1Frac(1, 1, _)   |  0.25
        // R1Frac(1, 2, _)   |  0.125
        // R1Frac(1, 3, _)   |  0.0625
        let U = R1Frac(1, 3, _);
        let P = X;     // |1⟩ basis state is convenient to experiment with R1 and R1Frac gates
        let n = 3;
        mutable counts = [0, size = 2^n];
        for _ in 1..100 {
            let res = PhaseEstimation(U, P, n);
            set counts w/= res <- counts[res] + 1;
        }
        for i in 0..2^n - 1 {
            if counts[i] > 0 {
                Message($"Eigenphase {IntAsDouble(i) / IntAsDouble(2^n)} - {counts[i]}%");
            }
        }
    }

    operation PhaseEstimation(
        U : Qubit => Unit is Ctl,
        P : Qubit => Unit,
        n : Int
    ) : Int {
        use (phaseRegister, eigenstate) = (Qubit[n], Qubit());
        P(eigenstate);
        ApplyToEach(H, phaseRegister);
        for k in 0..n - 1 {
            for _ in 1..1 <<< k {
                Controlled U([phaseRegister[k]], eigenstate);
            }
        }
        SwapReverseRegister(phaseRegister);
        Adjoint ApplyQFT(phaseRegister);

        Reset(eigenstate);
        return MeasureInteger(phaseRegister);
    }
}
```
