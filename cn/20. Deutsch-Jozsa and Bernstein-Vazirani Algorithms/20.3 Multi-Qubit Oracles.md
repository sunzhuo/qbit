# 多量子位预言机

在量子场景中，您正在使用的经典函数被实现为量子预言机 - 用作另一个算法的输入的“黑匣子”操作。此操作的实现方式使您不仅可以对单个输入执行计算，还可以对输入的叠加执行计算。

为了使预言机能够作用于量子态而不是经典值，整数输入 $x$ 以二进制 $x = (x_{0}, x_{1}, \dots, x_{N-1})$ 表示， 
并编码到 $N$-量子位寄存器：$\ket{\vec{x} } = \ket{x_{0} } \otimes \ket{x_{1} } \otimes \cdots \otimes \ket{x_{N-1} }$。
该函数的阶段预言机 $U_f$ 定义如下：

$$U_f \ket{\vec{x} } = (-1)^{f(x)} \ket{\vec{x} }$$

函数 $f$ 只能返回两个值，0 或 1，这分别导致没有相位变化或乘以相对相位 $-1$。

这种预言机对任何单一基础状态的影响并不是特别有趣：它只是添加了一个全局阶段，而这是您无法观察到的。然而，如果你将这个预言应用到基础状态的“叠加”上，它的效果就会变得显而易见。 
请记住，量子操作是线性的：如果您定义操作对基本状态的影响，您将能够使用其线性推断出其对叠加态（只是基本状态的线性组合）的影响。

让我们看看如何在 Q# 中将多位常量和平衡函数实现为相位预言机的几个示例。

1.$f(x) \equiv 0$

这是最容易实现的函数：if $f(x) \equiv 0$,

$$U_f \ket{x} \equiv (-1)^0 \ket{x} = \ket{x}$$

这意味着 $U_f$ 是一个恒等式 - 一个绝对不执行任何操作的转换！

2.$f(x) \equiv 1$

第二个常量函数稍微棘手一些： if $f(x) \equiv 1$

$$U_f \ket{x} \equiv (-1)^1 \ket{x} = - \ket{x}$$

现在 $U_f$ 是一个负恒等式，即将 $-1$ 的全局相位应用于状态的变换。 
许多算法只是忽略其中累积的全局相位，因为它是不可观察的。 
但是，如果您想要细致，可以使用 $R$ 门，它围绕给定轴执行给定旋转。 
当使用 `PauliI` 轴调用时，此操作将全局相位应用于给定的量子位。 
您可以使用任何量子位来应用此门，例如 `qs[0]`。

3.$f(x) = x \bmod 2$

$x$ 的二进制表示为 $x = (x_{0}, x_{1}, \dots, x_{N-1})$，最低有效位编码在最后一位（存储在输入数组的最后一个量子位中）。然后你可以将函数重写为

$$f(x) = x_{N-1}$$

让我们在预言效果表达式中使用它：

$$U_f \ket{x} = (-1)^{f(x)} \ket{x} = (-1)^{x_{N-1}} \ket{x} = \ket{x_{0} } \otimes \cdots \otimes \ket{x_{N-2} } \otimes (-1)^{x_{N-1}} \ket{x_{N-1}}$$

这意味着您只需要在实现中使用最后一个量子位：如果它是 $\ket{0}$，则不执行任何操作；如果它是 $\ket{1}$，则应用 $-1$ 的阶段。这正是 $Z$ 门的效果！

你可以写出如下的预言机酉：

$$U_f = \mathbb{1} \otimes \cdots \otimes \mathbb{1} \otimes Z$$

在下面的演示中，您将了解如何将三个多位函数实现为量子预言机，以及它们对量子态的影响。
之后，您将尝试自己实现另外两个功能的预言机！

来源：OracleImplementationDemo.qs

```qsharp
namespace Kata {
    import Std.Diagnostics.*;
    import Std.Math.*;

    operation PhaseOracle_Zero(x : Qubit[]) : Unit {
        // Do nothing...
    }

    operation PhaseOracle_One(x : Qubit[]) : Unit {
        // Apply a global phase of -1
        R(PauliI, 2.0 * PI(), x[0]);
    }

    operation PhaseOracle_Xmod2(x : Qubit[]) : Unit {
        let N = Length(x);
        // Array elements are indexed 0 through Length(x) - 1, inclusive.
        Z(x[N - 1]);
    }

    @EntryPoint()
    operation OracleImplementationDemo() : Unit {
        use qs = Qubit[2];
        Ry(2.0 * ArcCos(0.5), qs[0]);
        Ry(2.0 * ArcCos(0.6), qs[1]);
        Message("The state before oracle application:");
        DumpMachine();

        // Apply the oracle.
        // Experiment with using other oracles to see their behavior!
        // (Note that the -1 global phase might not show up in simulation)
        PhaseOracle_Xmod2(qs);

        Message("The state after oracle application:");
        DumpMachine();

        ResetAll(qs);
    }
}
```
