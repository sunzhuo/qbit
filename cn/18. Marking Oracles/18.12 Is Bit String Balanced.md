# 位串是否平衡？

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：** 
实现一个量子预言机，检查输入寄存器是否是平衡位串，即它是否恰好包含 $N/2$ $0$ 和 $N/2$ $1$。 $N$ 将为偶数。

例如，对于$N = 4$，基础状态$\ket{0011}$和$\ket{0101}$是平衡的，而$\ket{0010}$和$\ket{1111}$不是平衡的。

让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

让我们首先实现一个辅助操作，我们需要计算位字符串中 $1$ 位的数量。 
该函数将作用于存储整数的 $m$ 量子位寄存器，并以 $2^m$ 为模递增该整数。我们将其命名为 `Increment`，并以小端表示法存储输入。

考虑以二进制表示法递增经典数字的逻辑：

* 首先，我们翻转最低有效位。
* 如果最低有效位现在是$0$，则增加数字的其余部分（即翻转第二个最低有效位，检查它是否是$0$，并递归地应用相同的过程）。

我们可以交换这两个步骤，这样如果最低有效位是 $1$，我们首先增加数字的其余部分，然后翻转最低有效位。

我们如何将这些步骤转化为量子门？

* 经典条件语句转换为受控门（请记住，所有计算都必须在没有测量的情况下完成，因此如果输入寄存器处于叠加状态，则在计算后它也保持叠加状态）。我们可以使用 `Controlled` 函子来调用 `Increment` 的受控版本，第一个参数是控制量子位数组（最低有效位），第二个参数是传递给 `Increment` 的参数（其余位）。
* 翻转最低有效位对应于对其应用 $X$ 门。

实施此辅助操作后，解决任务就变得清晰了：

1. 分配辅助量子位来存储位串中的$1$位数。所需的量子位数量是位串长度 $N$ 的二进制表示法中的位数，您可以使用方便的 Q# 库函数 `BitSizeI` 进行计算。
2. 计算输入寄存器中$1$位的数量。为此，您需要使用 `Increment` 操作的受控变体（以该量子位为控制），为处于 $\ket{1}$ 状态的输入寄存器的每个量子位递增计数器数组。
3. 检查$1$位数是否恰好为$\frac{N}{2}$，如果使用`ApplyControlledOnInt`运算，则翻转目标量子位。
4. 像往常一样，在释放辅助量子位之前取消计算对它们所做的更改。

```qsharp
namespace Kata {
    import Std.Math.*;

    operation Oracle_Balanced(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        let N = Length(x);
        let log = BitSizeI(N);
        use inc = Qubit[log];
        within {
            for q in x {
                Controlled Increment([q], inc);
            }
        } apply {
            ApplyControlledOnInt(N / 2, X, inc, y);
        }
    }

    operation Increment(register : Qubit[]) : Unit is Adj + Ctl {
        if Length(register) > 1 {
            Controlled Increment([register[0]], register[1...]);
        }
        X(register[0]);
    }
}
```
