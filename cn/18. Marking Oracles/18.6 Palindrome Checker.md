# 回文检查器

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：** 
实现一个量子预言机，检查输入寄存器是否为回文，即，如果 $x$ 是回文，则实现函数 $f(x) = 1$，否则为 0。如果一个位串等于它的倒数，或者换句话说，它的第一位等于它的最后一位，它的第二位等于它的倒数第二位，等等，那么它就是回文。

例如，对于 $N = 3$，输入状态 $\ket{101}$ 是回文，而 $\ket{001}$ 不是。

让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

对于要成为回文的基础状态 $\ket{x_0 ... x_{N-1}}$，第一个量子位 $x_0$ 的状态必须与最后一个量子位 $x_{N-1}$ 的状态相同，第二个量子位 $x_1$ 的状态 - 与倒数第二个量子位 $x_{N-2}$ 的状态相同，依此类推上。
换句话说，我们需要计算量子位对的异或，如果所有这些异或都是 $0$，则基本状态是回文。

回想一下，要就地计算两个量子位状态的 XOR，您可以使用 $CNOT$ 门，其中一个作为控制，另一个作为目标。此后，目标量子位的状态将恰好是 XOR：

$$CNOT \ket{a}\ket{b} = \ket{a}\ket{a \oplus b}$$

我们可以用它来计算 $x_{N-1}$ 和 $x_0$（存储在量子位 `x[0]` 中）、$x_{N-2}$ 和 $x_1$（存储在量子位 `x[1]` 中）等的异或。最后，为了检查所有 XOR 是否都是 $0$，我们可以使用库操作 `ApplyControlledOnInt` 来应用受控零 $X$ 门，其中我们用于计算 XOR 的量子位作为控制，目标量子位作为目标。

请记住取消计算您对输入寄存器的量子位所做的更改。您可以使用自动处理取消计算的应用内模式轻松地完成此操作。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Oracle_Palindrome (x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        let N = Length(x);
        within {
            for i in 0 .. N / 2 - 1 {
                CNOT(x[N - 1 - i], x[i]);
            }
        } apply {
            ApplyControlledOnInt(0, X, x[... N / 2 - 1], y);
        }
    }    
}
```
