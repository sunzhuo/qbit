# 带否定的乘积函数

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。
3.长度为$N$的位向量$r$，表示为`Bool[]`。

**目标：** 
实现一个标记预言机，它计算以下函数：
$$f(x) = \bigoplus\limits_{i=0}^{N-1} \left(r_i x_i \oplus (1 - r_i) (1 - x_i) \right)$$

换句话说，对于每个基础状态 $\ket{x}$，如果 $f(x) = 1$，则翻转目标量子位 $\ket{y}$ 的状态，否则保持不变。
让输入寄存器中的量子位保持其开始时的状态。 
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

同样，这个问题与上一个问题类似。这次，每个输入量子位 `x[j]` 始终影响目标量子位的状态，并且匹配的经典输入 `r[i]` 指定其执行此操作的方式。 
如果位`r[i]`为`true`，并且输入量子位`x[j]`处于$\ket{1}$状态，则目标量子位的状态翻转；
否则，如果 `x[j]` 处于 $\ket{0}$ 状态，则翻转。 （你可以检查一下这是否正是公式所表达的 
$\left(r_i x_i \oplus (1 - r_i) (1 - x_i) \right)$ 评估为！）

这意味着我们需要修改上一个任务的解决方案，在 if 语句中添加第二个子句，以处理 `r[i]` 为 `false` 时的情况。在这种情况下，我们需要应用的门是零控制的 $X$，我们可以使用库操作 `ApplyControlledOnInt` 来实现。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Oracle_ProductWithNegation(x : Qubit[], y : Qubit, r : Bool[]) : Unit is Adj + Ctl {
        for i in 0 .. Length(x) - 1 {
            if r[i] {
                CNOT(x[i], y);
            } else {
                ApplyControlledOnInt(0, X, [x[i]], y);
            }
        }
    }
}
```
