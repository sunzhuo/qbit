# 位和能被 3 整除吗？

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：** 
实现一个量子预言机，检查位串中的位总和是否可以被 $3$ 整除。

例如，对于 $N = 3$，应标记的唯一基础状态是 $\ket{000}$ 和 $\ket{111}$。

让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

让我们首先考虑问题的一个更简单的版本。如何检查位数是否相等
到 $1$ 能被 $2$ 整除吗？

您可以迭代输入量子位并执行一系列 $CNOT$ 门，其中每个门为
控制和输出量子位作为目标。 $CNOT$ 对状态进行受控翻转这一事实确保了
这与取模 $2$ 位的总和相同，因此最后，如果总和模
$2$ 为 $0$。

您可以在此处使用相同的方法，但需要将加法模 $3$ 实现为初等运算。
为此，您使用一个带有两个量子位的计数器（首先存储 `sum` 量子位，然后存储 `carry`），并找出以某种方式更新它们的规则
每个增量允许从 $\ket{0_{sum}0_{carry}}$ 到 $\ket{1_{sum}0_{carry}}$ 到 $\ket{0_{sum}1_{carry}}$ 到 $\ket{0_{sum}0_{carry}}$。 （基本状态 $\ket{1_{sum}1_{carry}}$ 保持不变以保持变换单一，尽管我们永远不会在我们的解决方案中使用它。）一种方法如下：

1. 首先更新 `sum` 位。  
   如果 `carry` 为 $0$，则 `sum` 发生变化，否则保持不变。我们可以使用零控制 $X$ 门来实现这一点。
2. 然后，更新`carry`位。
   如果更新的 `sum` 位是 $0$，它会发生变化，这可以使用另一个受控零 $X$ 门来完成。

实施 `IncrementMod3` 操作后，您可以迭代输入量子位并以 $3$ 为模计算其中 $1$ 位的数量。如果两个计数器量子位最终都处于 $\ket{0}$ 状态，则 $1$ 位的数量可以被 $3$ 整除，并且您需要翻转目标量子位。

来源：Solution.qs

```qsharp
namespace Kata {
    operation Oracle_BitSumDivisibleBy3 (x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        use counter = Qubit[2];
        within {
            for q in x {
                Controlled IncrementMod3([q], counter);
            }
        } apply {
            ApplyControlledOnInt(0, X, counter, y);
        }
    }

    operation IncrementMod3 (counterRegister : Qubit[]) : Unit is Adj + Ctl {
        let sum = counterRegister[0];
        let carry = counterRegister[1];
        ApplyControlledOnInt(0, X, [carry], sum);
        ApplyControlledOnInt(0, X, [sum], carry);
    }    
}
```
