# 数字能被3整除吗？

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：** 
实现一个量子预言机，检查位串表示的数字是否可以被 $3$ 整除。
使用小端表示法将位串转换为整数，即最低有效位存储在 `x[0]` 中。

例如，对于 $N = 3$，应标记的基础状态为 $\ket{000} = \ket{0}$、$\ket{110} = \ket{3}$ 和 $\ket{011} = \ket{6}$。

让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

此问题与上一个问题类似：我们将计算以 $3$ 为模的表达式，因此我们将使用上一个任务中定义的 `IncrementMod3` 运算作为构建块。

然而，这次我们需要以 $3$ 为模来递增和递减数字。幸运的是，自减只是自增的伴随，Q# 编译器可以自动生成它，而无需您手动实现。

这次你需要一个额外的技巧：你需要计算该数字除以 $3$ 的余数，而不是仅仅计算 $1$ 位模 $3$ 的数量。您可以利用 $2$ 的连续幂依次给出余数 $1$ 和 $-1$ 的事实：$2^0 \equiv 1 \mod 3$、$2^1 = 2 \equiv -1 \mod 3$、$2^2 = 4 \equiv 1 \mod 3$、$2^3 = 8 \equiv -1 \mod 3$ 等。您从数字的最低有效位开始迭代，以量子位 0、2、4、... 作为控制执行计数器量子位的受控递增，并以量子位 1、3、5、... 作为控制执行计数器量子位的受控递减。

最后，仅当计数器量子位处于 $\ket{00}$ 状态时，数字本身才能被 $3$ 整除。

```qsharp
namespace Kata {
    operation Oracle_DivisibleBy3 (x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        use counter = Qubit[2];
        within {
            for i in 0 .. Length(x) - 1 {
                if i % 2 == 0 {
                    Controlled IncrementMod3([x[i]], counter);
                } else {
                    Controlled Adjoint IncrementMod3([x[i]], counter);
                }
            }
        } apply {
            ApplyControlledOnInt(0, X, counter, y);
        }
    }

    operation IncrementMod3 (counterRegister : Qubit[]) : Unit is Adj + Ctl {
        let sum = counterRegister[0];
        let carry = counterRegister[1];
        ApplyControlledOnInt(0, X, [carry], sum);
        ApplyControlledOnInt(0, X, [sum], carry);
    }    
}
```
