# 多数函数

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。 $N$ will be one of the values $3, 5, 7$.
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。

**目标：** 
实现一个计算多数函数的量子预言机，即，如果位串中的大多数位是 $1$，则为 $f(x) = 1$，否则为 $0$。
    
例如，对于 $N = 3$，基本状态 $\ket{001}$ 和 $\ket{000}$ 的多数函数是 $0$，对于 $\ket{101}$ 和 $\ket{111}$ 是 $1$。

让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

此任务还将依赖于对位字符串中的 $1$ 位进行计数，我们将使用与上一个任务中相同的方法。然而，这次我们需要检查的条件有所不同：我们不是检查 $1$ 位数是否是固定数字，而是检查它是否大于常数 $\frac{N - 1} / 2$。理想情况下，我们希望在不实现将存储在量子位数组中的整数与常量进行比较的一般逻辑的情况下做到这一点。

为了找到适合我们案例的快捷方式，让我们看看我们想要将目标量子位翻转为不同 $N$ 值的位串。

您可以注意到我们需要标记 $N = 3$ 和 $N = 7$ 的位串中的模式：它们都是最高有效位等于 $1$ 的可能位串，因此可以使用受控 $X$ 门以单个量子位作为控制来非常轻松地标记它们 - 量子位存储计数数字的最高有效位！

不幸的是，这种模式在 $N = 5$ 中被打破，因为我们需要用等于 $0$ 的最高有效位来标记位串 $110$。但这很容易通过一点技巧来修复：我们只需使用具有两个最低有效位作为控制的 $CCNOT$ 门单独标记该位串，利用该位串是唯一满足此条件的事实。

> 请记住，这是一个依赖于我们输入中少量量子位的黑客行为。在一般情况下，我们希望实现正确的逻辑来将位数与常量进行比较（我们将在后面的 katas 之一中学习如何做到这一点）。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Math.*;

    operation Oracle_Majority(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        let N = Length(x);
        let log = BitSizeI(N);
        use inc = Qubit[log];
        within {
            for q in x {
                Controlled Increment([q], inc);
            }
        } apply {
            CNOT(inc[log-1], y);
            if N == 5 {
                CCNOT(inc[0], inc[1], y);
            }
        }
    }

    operation Increment(register : Qubit[]) : Unit is Adj + Ctl {
        if Length(register) > 1 {
            Controlled Increment([register[0]], register[1...]);
        }
        X(register[0]);
    }
}
```
