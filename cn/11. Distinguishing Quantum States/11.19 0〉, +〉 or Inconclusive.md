# |0〉、|+〉 或不确定？

**输入：** 保证处于 $\ket{0}$ 或 $\ket{+}$ 状态的量子位。
    
**输出：** 
* 0 如果量子位处于 $\ket{0}$ 状态， 
* 1 如果处于 $\ket{+}$ 状态，
* -1 如果您无法决定，即“不确定”的结果。

您的解决方案：

* 决不应该错误地给出 0 或 1 答案（即将 $\ket{0}$ 识别为 1 或 $\ket{+}$ 识别为 0），
* 将被多次调用，每次都会以相同的概率选择其中一个状态，
* 在最多 80% 的情况下可能会给出不确定的 (-1) 答案，
* 必须在至少 10% 的情况下正确地将 $\ket{0}$ 状态识别为 0，
* 必须在至少 10% 的情况下正确地将 $\ket{1}$ 状态识别为 1。
    
操作结束时量子位的状态并不重要。
    
> 这项任务是明确的国家歧视的一个例子。

## 解决方案注释

可以通过在计算基础和哈达玛基础之间随机选择测量基础来获得一种简单的策略，该策略以 0.75 的概率给出不确定的结果，并且在产生结论性结果的情况下永远不会出错。
    
请注意，在标准基础上测量时，状态 $\ket{0}$ 将始终导致结果“0”，而状态 $\ket{+}$ 将分别以 $\frac12$ 的概率导致结果“0”和“1”。这意味着如果我们测量“1”，我们可以确定状态为 $\ket{+}$。
    
类似的论点适用于我们在哈达玛基础上进行测量的情况，其中 $\ket{0}$ 可以导致“+”和“-”结果，而 $\ket{+}$ 始终导致“+”。那么如果我们测量到“-”，我们可以肯定地得出结论，状态是$\ket{0}$。

由于四种情况中的每一种都以 25% 的概率发生，因此总体而言，该策略最终以 12.5% 的概率正确识别 $\ket{0}$ 和 $\ket{+}$ 状态，并以 75% 的概率给出不确定的结果。

> 在 Q# 中实现 Hadamard 基础测量的最简单方法是应用 H 门，然后在计算基础上进行常规测量。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Random.*;
    operation IsQubitZeroPlusOrInconclusive(q : Qubit) : Int {
        // Pick a random basis
        let basis = DrawRandomInt(0, 1);
        if basis == 0 {
            // use standard basis
            let result = M(q);
            // result is One only if the state was |+⟩
            return result == One ? 1 | -1;
        } else {
            // use Hadamard basis
            H(q);
            let result = M(q);
            // result is One only if the state was |0⟩
            return result == One ? 0 | -1;
        }
    }
}
```
