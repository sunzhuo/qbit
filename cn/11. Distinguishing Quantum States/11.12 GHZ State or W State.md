# GHZ 状态还是 W 状态？

**输入：** $N \ge 2$ 量子位（存储在长度为 $N$ 的数组中）保证处于 GHZ 状态（$\ket{0...0}$ 和 $\ket{1...1}$ 状态的偶数叠加）或处于我们在上一个任务中看到的 W 状态。 （例如，对于 $N = 3$ GHZ 状态为 $\frac1{\sqrt2}(\ket{000} + \ket{111})$。

**输出：**
* 0 如果量子位处于 GHZ 状态，
* 1 如果他们处于 W 状态。

操作结束时量子位的状态并不重要。

## 解决方案注释

正如我们在上一个任务中所看到的，形成 W 状态的每个基础状态将恰好有一个处于 $\ket{1}$ 状态的量子位。形成 GHZ 状态的基础状态将具有 $\ket{1}$ 状态中的所有量子位或 $\ket{0}$ 状态中的所有量子位。

这意味着，如果我们计算在 `One` 状态下测量的量子位数量，我们将在 W 状态下得到 1，在 GHZ 状态下得到 0 或 $N$。该代码最终与上一个任务几乎相同（事实上，您可以使用这个确切的代码来解决上一个任务）。

来源：Solution.qs

```qsharp
namespace Kata {
    operation GHZOrWState(qs : Qubit[]) : Int {
        mutable countOnes = 0;

        for q in qs {
            if M(q) == One {
                set countOnes += 1;
            }
        }

        return countOnes == 1 ? 1 | 0;
    }
}
```
