# 评估一个子句

**输入：**

1. $N$ 量子位处于任意状态 $\ket{x}$（输入/查询寄存器）。
2. 处于任意状态 $\ket{y}$ 的量子位（输出/目标量子位）。
3. 一组元组 `clause`，描述 SAT 问题实例 $clause(x)$ 的一个子句。
   每个元组都是一个 `(Int, Bool)` 对，描述该子句的一个组成部分：
   - 第一个元素是变量 $x_j$ 的索引 $j$， 
   - 如果变量作为自身包含 ($x_j$)，则第二个元素为 `true`，如果作为否定包含 ($\neg x_j$)，则第二个元素为 `false`。

例如，子句 $x_0 \vee \neg x_1$ 可以表示为 `[(0, true), (1, false)]`。

**目标：**
实现一个评估子句 $clause(x)$ 的量子预言机。
    
让输入寄存器中的量子位保持其开始时的状态。
您的解决方案应该适用于叠加输入，而不是使用任何测量。

## 解决方案注释

此任务涉及评估一个子句，该子句是输入 $x$ 中编码的否定变量和非否定变量的析取 (OR)。 
这可以分两步完成：

1. 首先，翻转 `clause` 中被取反的量子位（然后撤消此操作）。  
   为此，当且仅当子句具有 `(j, false)` 形式的项时，将 $X$ 门应用于量子位 $j$。
2. 使用 $U_{or}$ 酉（由运算 `Oracle_Or` 实现）来计算子句。  
   为此，您需要首先构造一个数组 `clauseQubits` - 作为子句中的否定或非否定变量包含的所有量子位。然后，您可以将 `Oracle_Or` 运算应用于量子位 `clauseQubits` 和 `y`。

请注意，`Oracle_SATClause` 的实现应该是可伴随的，并且能够依赖 Q# 编译器生成此操作的伴随变体是很好的。
这就是为什么该解决方案使用库函数 `Mapped` 而不是涉及操作可变变量的经典计算 - 这些变量必须移动到单独的函数中，从而使代码变得更加庞大。

来源：Solution.qs

```qsharp
namespace Kata {
    import Std.Arrays.*;
    operation Oracle_SATClause(x : Qubit[], y : Qubit, clause : (Int, Bool)[]) : Unit is Adj + Ctl {
        let clauseQubits = Mapped((ind, _) -> x[ind], clause);
        within {
            for (ind, positive) in clause {
                if not positive {
                    X(x[ind]);
                }
            }
        } apply {
            Oracle_Or(clauseQubits, y);
        }
    }

    operation Oracle_Or(x : Qubit[], y : Qubit) : Unit is Adj + Ctl {
        ApplyControlledOnInt(0, X, x, y);
        X(y);
    }
}
```
